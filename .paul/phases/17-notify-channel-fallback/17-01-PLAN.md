<objective>
## Goal
"Try webpush, if not opened in N minutes, send email." Multi-channel durable delivery without duplicate notifications if the primary channel succeeded.

## Purpose
Close the gap where a failed/ignored webpush would never fall back to email. Uses the already-working `scheduled_at` infrastructure from Phase 16 — no new worker needed.

## Output
- `12_fct_notify_templates.fallback_chain JSONB` — e.g. `[{"channel_id": 1, "wait_seconds": 300}]`.
- Transactional Send creates the primary delivery immediately + N scheduled deliveries (one per fallback step).
- Each sender, before sending, checks if the primary (same template_id + recipient + earlier created_at) has reached status `opened` — if so, marks self `unsubscribed` with `failure_reason="superseded_by_primary"` and returns early.
- Templates view exposes the new column.
- Test proves the superseded path.
</objective>

```gherkin

When POST /v1/notify/send is called with channel_code="webpush"
Then two delivery rows exist:
  (a) channel_code="webpush", scheduled_at IS NULL (immediate)
  (b) channel_code="email",   scheduled_at ≈ now + 60s
```

## AC-3: Superseded-by-primary skip
```gherkin
Given a primary webpush delivery with status='opened'
  And a fallback email delivery with scheduled_at <= now()
When the email sender processes the fallback
Then its status becomes 'unsubscribed' with failure_reason startswith 'superseded_'
  And no SMTP call is made
```
