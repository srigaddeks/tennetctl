<objective>
## Goal
Close the 🔴 Notify Subscriptions admin gap by adding inline **edit** paths to `/notify/settings` for Subscriptions, SMTP Configs, and Template Groups. Today only create+delete exist — operators can't tweak a subscription without deleting and recreating it.

## Purpose
Per COVERAGE-MATRIX.md, subscriptions has no admin page; on inspection, `/notify/settings` does list all three resources, but edit is missing end-to-end. "Subscribe to critical signals without UI is painful" — adding edit UX without delete+recreate unblocks day-to-day admin work.

## Output
- New type exports: `NotifySMTPConfigUpdate`, `NotifyTemplateGroupUpdate`, `NotifySubscriptionUpdate`
- New update hooks for all three resources
- Edit modals in `/notify/settings`; rows expose an Edit button alongside Delete
</objective>
