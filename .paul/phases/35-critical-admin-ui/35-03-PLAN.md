<objective>
## Goal
Ship a single-pane System Health admin surface: enhance `/health` with rich module + infrastructure stats, build a `/system/health` admin page that renders them with per-subsystem status badges.

## Purpose
COVERAGE-MATRIX.md flagged "System Health dashboard" as 🔴 — today there is no place to see which modules booted, how deep the pool is, whether vault + NATS + catalog are healthy. Today's `/health` returns a hard-coded `{"status": "healthy"}`, so it cannot back an admin UI.

## Output
- Enhanced `/health` endpoint returning structured subsystem report
- New hook `use-system-health.ts`
- New admin page at `/system/health`
- Type `SystemHealthReport` in `api.ts`
</objective>

## AC-1: Rich /health payload
```gherkin
Given the backend is running
When GET /health is called
Then the response envelope contains:
  - app.version (string)
  - modules.enabled (string[])
  - modules.available (string[])
  - pool.size / pool.free / pool.busy (integers; -1 allowed if pool missing)
  - vault.enabled (bool), vault.ok (bool)
  - catalog.features / catalog.sub_features / catalog.nodes (counts from last upsert report, or nulls if unavailable)
  - nats.configured (bool), nats.url_host (string without scheme/credentials)
  - db.ok (bool, result of SELECT 1)
  - checked_at (ISO timestamp)
And the endpoint continues to return 200 even if a subsystem is degraded (per-subsystem `ok` flag conveys state)
```

## AC-2: System Health page renders report
```gherkin
Given the admin visits /system/health
When the page loads
Then each subsystem (database, vault, catalog, nats, modules) renders as a card with:
  - Subsystem name
  - Status Badge (emerald for ok, amber for degraded, red for down, zinc for disabled/not-configured)
  - Key metrics inline (e.g. "pool 5/10 in use", "12 features · 67 nodes")
And a "Refresh" button re-fetches /health
And a timestamp shows when the data was checked
```

## AC-3: Module inventory visible
```gherkin
Given the admin is on /system/health
When the Modules card renders
Then every module from MODULE_ROUTERS shows a row with:
  - Module name
  - Enabled Badge (emerald "on" / zinc "off")
And the Enabled rows match TENNETCTL_MODULES output
```
