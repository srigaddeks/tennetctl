## AC-1: Navigable detail page
```gherkin
Given a workspace exists with id W
When an admin visits /iam/workspaces/W
Then the page renders slug, display_name, org, is_active badge, created_at
And a "Back to workspaces" link returns to the list
```

## AC-2: Inline edit + soft-delete
```gherkin
Given the admin is on the workspace detail page
When they change display_name or slug and click Save
Then PATCH /v1/workspaces/{id} fires with only the dirty fields
And a success toast displays

Given the admin clicks Delete
When they confirm
Then DELETE /v1/workspaces/{id} fires
And they are redirected to /iam/workspaces with a success toast
```

## AC-3: Member list + add + remove
```gherkin
Given the workspace has members
When the detail page loads
Then the Members section lists each workspace_member with the user's email + display_name and a Remove button

Given the admin picks a user from the "Add member" selector
When they click Add
Then POST /v1/workspace-members fires with { user_id, workspace_id }
And the member appears in the list

Given the admin clicks Remove on a membership
When they confirm
Then DELETE /v1/workspace-members/{membership_id} fires
And the row disappears from the list
```

## AC-4: List page links, no drawer
```gherkin
Given the admin is on /iam/workspaces
When they click any row
Then the browser navigates to /iam/workspaces/{id}
And the WorkspaceDetailDrawer component is no longer rendered or imported
```
