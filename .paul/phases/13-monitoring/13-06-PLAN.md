# Plan 13-06 — Monitoring UI: Log Explorer + Metric Dashboards + Trace Waterfall + Robot E2E

**Phase:** 13 — Monitoring
**Goal:** Ship the full visual layer. Log explorer with live tail (SSE), metric dashboards with draggable panels, distributed-trace waterfall view, saved-query browser, DSL filter builder. Robot E2E across all four flows.

---

<objective>

## Goal
Make the monitoring data usable by humans without psql. Every page is data-dense, keyboard-navigable, and built on the 13-05 query DSL.

## Purpose
Without UI, internal monitoring is a developer-only tool. Dashboards + log explorer + trace waterfall are the visible replacement for Grafana + Loki + Tempo + Jaeger.

## Output
- 5 pages: /monitoring (overview), /monitoring/logs, /monitoring/metrics, /monitoring/traces, /monitoring/traces/[id], /monitoring/dashboards, /monitoring/dashboards/[id]
- 9 reusable components (log-explorer, log-live-tail, metrics-chart, metric-picker, trace-waterfall, dashboard-grid, panel, timerange-picker, dsl-filter-builder)
- Dashboards schema (fct_monitoring_dashboards + fct_monitoring_panels) + CRUD sub-feature
- Live tail via SSE (`GET /v1/monitoring/logs/tail`)
- 4 Robot E2E suites

</objective>

## AC-3: /monitoring/logs explorer
```gherkin
Given logs exist
When user visits /monitoring/logs
Then page shows:
  - Timerange picker (last 15m/1h/24h/7d + custom)
  - Severity filter (multi-select)
  - Body text search (debounced 300ms)
  - Attribute filter builder (DSL-backed — key, op, value rows)
  - Results table (time, severity colored, service, body truncated, trace_id linked to /traces/[id])
  - Row expand → full attributes + resource_attributes JSON
  - Pagination via cursor (scroll-triggered load-more)
  - "Live tail" toggle → switches to SSE stream, auto-scroll, pauses on user scroll
```

## AC-4: /monitoring/metrics dashboards
```gherkin
Given metrics are registered
When user visits /monitoring/metrics
Then:
  - metric-picker lists registered metrics (search, filter by kind)
  - selecting a metric shows chart preview with label-filter builder + aggregate + bucket picker
  - chart renders via Recharts (or similar; pick one and stick)
  - histograms render p50/p95/p99 as three lines
  - counters render rate-per-second (auto-computed via `rate` aggregate)
  - timerange-picker shared across the page
  - "Add to dashboard" button opens dialog to select existing dashboard + panel size, or create new
```

## AC-5: /monitoring/traces search + waterfall
```gherkin
Given spans exist
When user visits /monitoring/traces
Then table shows recent traces (trace_id, root service, root span name, duration, has_error flag, timestamp)
And filters: service, name contains, duration range, error-only toggle
When user clicks a trace row or visits /monitoring/traces/[traceId]
Then waterfall view renders:
  - Each span as a horizontal bar, indented by parent-child hierarchy
  - Bar width proportional to duration
  - Color by service
  - Click span → side panel with attributes, events, links
  - Keyboard nav j/k through spans
```

## AC-6: /monitoring/dashboards
```gherkin
Given dashboards exist
When user visits /monitoring/dashboards
Then list view: cards per dashboard (name, desc, owner, shared flag, panel count, last modified)
When user visits /monitoring/dashboards/[id]
Then draggable/resizable grid of panels (react-grid-layout)
And each panel renders based on panel_type using the saved DSL
And edit mode: add panel, delete panel, drag/resize, edit title + DSL
And Save button persists layout JSONB + any panel DSL changes
```

## AC-7: Live tail SSE
```gherkin
Given GET /v1/monitoring/logs/tail with ?filter=<b64 DSL>
Then server holds SSE connection
And polls evt_monitoring_logs every 1s (or LISTEN/NOTIFY per 13-07 wiring) for rows newer than last yielded id
And yields `data: {log}\n\n` per new record
And connection closes cleanly on client disconnect
And backpressure: if client slow, drop oldest buffered events + send a "dropped: N" SSE event
```
