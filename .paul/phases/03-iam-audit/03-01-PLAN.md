<objective>
## Goal
Create all IAM database tables in the `"03_iam"` schema: dim lookup tables, EAV foundation (dim_entity_types, dtl_attr_defs, dtl_attrs), fct entity tables (orgs, workspaces, users, roles, groups, applications), and lnk join tables — using the distributed migration structure with seed YAML files for all dim data.

## Purpose
Phase 3+ verticals (Orgs, Workspaces, Users) need these tables to exist before any repo/service/route code can be written. This plan is schema-only — no Python code, no routes, no UI. Getting the schema right here prevents costly migrations later.

- 4 seed YAML files in the same `seeds/` directory
- All migrations applied + seeds loaded against local Postgres (port 5434)
- Files move to `01_migrated/` after apply
</objective>

## Database Conventions (from CLAUDE.md + rules/database.md)
Table types:
- `dim_*` (01–09): lookup enums, SMALLINT PK, seeded via YAML, never mutated. Mandatory cols: id, code, label, description, deprecated_at.
- `fct_*` (10–19): entity identity, VARCHAR(36) UUID v7 PK. No strings, no JSONB. Mandatory cols: id, is_active, is_test, deleted_at, created_by, updated_by, created_at, updated_at.
- `dtl_*` (20–29): EAV attributes. UUID v7 PK.
- `lnk_*` (40–59): many-to-many, immutable. UUID v7 PK. Mandatory: id, org_id, created_by, created_at. NO updated_at.
- `evt_*` (60–79): append-only. NO updated_at, NO deleted_at.

Critical:
- NEVER add business columns to fct_* (no email, no name, no slug on users)
- Slugs (org.slug, workspace.slug) ARE on fct tables — they're identity discriminators, not business attrs
- Strings, names, labels go in dtl_attrs via the EAV pattern
- NEVER Postgres ENUMs — use dim_* tables
- TIMESTAMP not TIMESTAMPTZ — app always passes UTC
- CURRENT_TIMESTAMP not now()
- deleted_at TIMESTAMP — not is_deleted BOOLEAN
- Partial unique index instead of NULLS NOT DISTINCT
- All constraints explicitly named: pk_, fk_, uq_, chk_, idx_
- COMMENT ON every table and column
- updated_at set by app in every UPDATE — no triggers
- lnk_* and evt_* have NO updated_at
</context>
