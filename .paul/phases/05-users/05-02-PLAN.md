- 4 catalog nodes — iam.memberships.{org,workspace}.{assign,revoke} — all effect, all emit audit
- Parent-entity validation via run_node("iam.users.get", ...), run_node("iam.orgs.get", ...), run_node("iam.workspaces.get", ...)
- 4 pytest integration tests — assign/revoke for org + workspace, missing-parent envelopes, list filters
</objective>

<context>
## Lnk tables (Phase 3)
- `40_lnk_user_orgs` (id, user_id, org_id, created_by, created_at) — UNIQUE(user_id, org_id); FKs to fct_users + fct_orgs
- `41_lnk_user_workspaces` (id, user_id, workspace_id, org_id, created_by, created_at) — UNIQUE(user_id, workspace_id); FKs to fct_users + fct_workspaces + fct_orgs

Lnk tables are immutable per CLAUDE.md — no `updated_at`, no `deleted_at`. Revoke = hard DELETE.
Membership records emit audit on create + delete (creation = grant, delete = revoke); both are auditable actions.
org_id in lnk_user_workspaces is the tenant discriminator, derived from the workspace's org_id at assignment time.

## Cross-sub-feature validation
- Validate user exists → `run_node("iam.users.get", ctx, {id})`
- Validate org exists → `run_node("iam.orgs.get", ctx, {id})`
- Validate workspace exists → `run_node("iam.workspaces.get", ctx, {id})` — returns org_id too so we can fill lnk_user_workspaces.org_id
</context>
