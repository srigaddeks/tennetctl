## User-specific shape
- `fct_users`: id, **account_type_id** SMALLINT FK to dim_account_types, is_active, is_test, audit cols
- `dim_account_types`: seeded 4 rows (email_password, magic_link, google_oauth, github_oauth)
- EAV attrs on `dtl_attrs` entity_type_id=3: email, display_name, avatar_url (all text)
- `v_users` resolves account_type as TEXT code (hides FK id); pivots the 3 attrs
- No uniqueness constraint on email in v1 — accepted; real email-based auth uniqueness lands in Phase 7
</context>
