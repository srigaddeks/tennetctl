<objective>
## Goal
Ship `tennetctl-py` — the Python half of the unified SDK — with a working transport layer (bearer auth, retry, envelope parsing, typed errors) and a complete `client.auth` module that wraps the existing IAM auth + sessions + api-keys HTTP endpoints.

## Purpose
Unified SDK is the v0.2.x spine. Every capability module (flags, iam, audit, notify, metrics, logs, traces, vault, catalog) will layer on top of the same transport + client shape delivered here. Every admin portal page (v0.2.4) will dogfood this SDK. Getting the contract right now avoids re-litigating it 8 times.

## Output
- `sdk/python/tennetctl/` importable as `tennetctl` (`pip install -e sdk/python`)
- `Tennetctl(base_url, api_key=...)` instance with a `.auth` namespace that covers signin / signout / me / sessions / api_keys
- pytest coverage ≥80% on `_transport.py` and `auth.py`
- SDK quickstart at `03_docs/00_main/09_guides/sdk-quickstart.md`
- 28-01-SUMMARY.md documenting the SDK contract so 28-02 (TS) can mirror it exactly
</objective>

## AC-2: Transport enforces bearer auth, envelope parsing, and retry
```gherkin
Given a Tennetctl client with api_key="nk_abc.def"
When any SDK method issues a request
Then the request has header "Authorization: Bearer nk_abc.def"
And 2xx responses with {"ok": true, "data": {...}} return .data to the caller
And 4xx/5xx responses with {"ok": false, "error": {"code": "X", "message": "Y"}} raise the mapped typed error (AuthError on 401/403, RateLimitError on 429, TennetctlError otherwise)
And transient errors (connect/read timeout, 502/503/504) retry with exponential backoff up to 3 attempts total
And non-transient errors (400/401/403/404/409/422) do not retry
```

## AC-3: client.auth covers signin / signout / me / sessions / api_keys
```gherkin
Given the backend is running
When client.auth.signin(email=e, password=p) succeeds
Then a session token is stored on the client and returned as a typed response
When client.auth.me() is called on an authenticated client
Then the current user is returned as a typed response
When client.auth.sessions.list() is called
Then a list of the caller's active sessions is returned
When client.auth.sessions.revoke(session_id) is called
Then the session is revoked server-side (DELETE /sessions/{id})
When client.auth.api_keys.create(name="test", scopes=["audit:read"]) is called
Then the response includes the one-time token
When client.auth.api_keys.list() / .revoke(id) / .rotate(id) are called
Then each wraps the matching backend endpoint and returns typed responses
When client.auth.signout() is called
Then POST /signout is issued and the stored session token is cleared on the client
```

  <action>
    1. Create `sdk/python/pyproject.toml` with:
       - `name = "tennetctl"`, `version = "0.1.0"`, `requires-python = ">=3.10"`
       - Runtime deps: `httpx>=0.27`, `pydantic>=2.7` (no more — keep surface thin)
       - Dev deps: `pytest`, `pytest-cov`, `pytest-asyncio`, `respx` (httpx mock)
       - Build backend: `hatchling`
       - Package layout: `tennetctl/` source dir

    2. `tennetctl/errors.py`:
       - `class TennetctlError(Exception)` with `code: str`, `message: str`, `status: int | None`
       - Subclasses: `AuthError` (401/403), `RateLimitError` (429), `NetworkError` (connection/timeout before response), `ServerError` (5xx)
       - `map_error(status: int, envelope: dict) -> TennetctlError` factory that picks the right subclass from HTTP status + server-sent error code

    3. `tennetctl/_transport.py`:
       - `class Transport` (async) wrapping `httpx.AsyncClient(base_url=..., timeout=30.0)`
       - Constructor accepts `base_url`, `api_key?`, `session_token?`; builds Authorization header (Bearer api_key preferred; falls back to Cookie-based session_token if provided — keep both paths for dev/test)
       - `async def request(method, path, *, json=None, params=None) -> dict`
         - Issues request, expects response envelope `{ok: bool, data?, error?}`
         - On `ok=true` → returns `data`
         - On `ok=false` → raises `map_error(status, envelope)`
         - On connection/read error → raises `NetworkError`
       - Retry policy: retry on `NetworkError` and status in {502, 503, 504}; exponential backoff (0.5s, 1s, 2s); max 3 attempts; do NOT retry on 4xx
       - Close method for cleanup; also support `async with` via `__aenter__` / `__aexit__`

    4. `tennetctl/client.py`:
       - `class Tennetctl`:
         - `__init__(base_url: str, *, api_key: str | None = None, session_token: str | None = None)`
         - Lazy-creates `Transport`; exposes `self._t` internally
         - `self.auth = Auth(self._t, self)` — Auth module defined in Task 2; import inside method to avoid circular import on module init
         - `async def close(self)` delegates to transport close
       - Public symbols re-exported from `tennetctl/__init__.py`: `Tennetctl`, `TennetctlError`, `AuthError`, `RateLimitError`, `NetworkError`, `ServerError`

    5. `sdk/python/README.md`: 30-line quickstart (install via `pip install -e .`, basic signin example, link to `03_docs/00_main/09_guides/sdk-quickstart.md`)

    Avoid:
    - Any sync code paths — SDK is async-first
    - Bundling pydantic models for every response; keep return types as TypedDict / plain dict in this plan. Typed Pydantic models land in 29-01 where the response surface is bigger
    - Caching at the transport level — SWR caching is a module-level concern (flags evaluate cache in 29-01)
    - SDK never imports from `backend/` — treat it as a pure HTTP client
  </action>

  </verify>

<task type="auto">
  <name>Task 2: client.auth module — signin/signout/me/sessions/api_keys</name>
  <files>
    sdk/python/tennetctl/auth.py

  <action>
    1. Discover the actual mounted URL prefixes by reading `backend/main.py` or the IAM feature manifest — the routes.py files shown in context are relative. Expect `/v1/iam/auth/*`, `/v1/iam/sessions/*`, `/v1/iam/api-keys/*` (or similar). Use the real paths — do not guess.

    2. Structure — submodules are attribute namespaces, not separate classes per call:
       ```python
       class Auth:
           def __init__(self, transport, client): self._t = transport; self._c = client
           async def signin(self, email, password) -> dict: ...
           async def signout(self) -> None: ...
           async def me(self) -> dict: ...
           @property
           def sessions(self) -> "Sessions": return Sessions(self._t)
           @property
           def api_keys(self) -> "ApiKeys": return ApiKeys(self._t)

       class Sessions:
           async def list(self) -> list[dict]: ...
           async def get(self, session_id: str) -> dict: ...
           async def revoke(self, session_id: str) -> None: ...

       class ApiKeys:
           async def list(self) -> list[dict]: ...
           async def create(self, name: str, scopes: list[str]) -> dict: ...   # returns {id, token_shown_once, ...}
           async def revoke(self, key_id: str) -> None: ...
           async def rotate(self, key_id: str) -> dict: ...                    # returns {id, token_shown_once}
       ```

    3. `signin` semantics:
       - POST credentials; on success, extract the session token from the response (Set-Cookie or body payload depending on backend shape)
       - If the client was constructed without `api_key`, store the session token on `self._c._session_token` AND on `self._t` so subsequent calls are authenticated
       - Return the raw `data` envelope to the caller (includes user + session info)

    4. `signout` semantics:
       - POST to signout endpoint; clear `self._c._session_token` and `self._t._session_token`; return `None`

    5. `me` calls GET /me and returns user dict

    6. All methods propagate typed errors from the transport — no `try/except` swallowing

    Avoid:
    - Adding OAuth (google/github) in this plan — those are browser-redirect flows, not SDK-native; they land in 28-02 (TS has the browser context)
    - Adding magic-link / OTP / passkey methods — those are v0.2.2+ when the full auth surface matures
    - Building response Pydantic models — keep return types as dicts for now
  </action>
