# Delivery Routes + Riders + Runs

Schema:
- `dim_rider_roles` (SMALLINT PK, code UNIQUE, name, deprecated_at) — UNIVERSAL seed: owner, contractor, employee, partner, gig
- `fct_delivery_routes` (UUID PK, tenant_id, kitchen_id FK, name, slug, area TEXT, target_window_start TIME, target_window_end TIME, status TEXT CHECK in ('active','paused','decommissioned') DEFAULT 'active', properties JSONB NOT NULL DEFAULT '{}', audit/ts/soft-delete) — UNIQUE (tenant_id, slug) WHERE deleted_at IS NULL
- `lnk_route_customers` (UUID PK, tenant_id, route_id FK, customer_id FK, sequence_position INT NOT NULL, created_at TIMESTAMP, created_by UUID — IMMUTABLE per lnk_ rule, NO updated_at, NO deleted_at; HARD delete to remove)
- `fct_riders` (UUID PK, tenant_id, user_id UUID NULL (link to tennetctl iam user when staff has account; nullable for contractors w/o accounts), name TEXT NOT NULL, phone TEXT, role_id SMALLINT FK to dim_rider_roles, vehicle_type TEXT (bike/scooter/van/walking), license_number TEXT, status TEXT CHECK in ('active','inactive','suspended') DEFAULT 'active', properties JSONB, audit/ts/soft-delete)
- `fct_delivery_runs` (UUID PK, tenant_id, route_id FK, rider_id FK, run_date DATE NOT NULL, status TEXT CHECK in ('planned','in_transit','completed','cancelled') DEFAULT 'planned', started_at TIMESTAMP NULL, completed_at TIMESTAMP NULL, total_stops INT NOT NULL DEFAULT 0, completed_stops INT NOT NULL DEFAULT 0, missed_stops INT NOT NULL DEFAULT 0, notes TEXT, properties JSONB, audit/ts/soft-delete) — partial unique (tenant_id, route_id, run_date) WHERE deleted_at IS NULL AND status != 'cancelled' (one active run per route per date)
- `dtl_delivery_stops` (UUID PK, tenant_id, delivery_run_id FK, customer_id FK, sequence_position INT NOT NULL, scheduled_at TIMESTAMP, actual_at TIMESTAMP NULL, status TEXT CHECK in ('pending','delivered','missed','customer_unavailable','cancelled','rescheduled') DEFAULT 'pending', photo_vault_key TEXT, signature_vault_key TEXT, notes TEXT, properties JSONB, audit/ts/soft-delete) — partial unique (delivery_run_id, customer_id) WHERE deleted_at IS NULL

Views: v_delivery_routes (JOIN kitchen + COUNT customers via lnk); v_delivery_runs (JOIN route + rider + computed completion_pct); v_delivery_stops (JOIN customer + computed time-since-scheduled).

Sub-features:
- `90_delivery_routes` — routes CRUD + route customers attach/detach (read all, attach single, attach bulk reorder, detach single)
- `93_riders` — riders CRUD + dim_rider_roles read
- `95_delivery_runs` — runs CRUD + stops CRUD + auto-generate-stops endpoint (POST /runs/{id}/generate-stops takes route's lnk_route_customers + creates dtl_delivery_stops in sequence)

Endpoints:
- `/v1/somaerp/delivery/routes` — list + CRUD; `/{id}/customers` — list + POST {customer_id, sequence_position} + DELETE /{customer_id} + POST /reorder ({customer_ids: [in_order]}) for bulk reorder
- `/v1/somaerp/delivery/rider-roles` (read), `/v1/somaerp/delivery/riders` — list + CRUD
- `/v1/somaerp/delivery/runs` — list + CRUD + state transitions (planned→in_transit→completed | cancelled); POST /{id}/generate-stops; GET /{id}/stops; PATCH /{id}/stops/{stop_id} (mark delivered/missed/photo); GET /board?date= today's runs grouped by kitchen.

Status transition emits audit `somaerp.delivery.runs.{started,completed,cancelled}` and `somaerp.delivery.stops.{delivered,missed,rescheduled}`.

Frontend:
- /delivery landing (3 cards: Routes, Riders, Today's Runs)
- /delivery/routes list + new + [id] (route detail with customer reorder UI — drag handle disabled in v0; just position number editable)
- /delivery/riders list + new
- /delivery/runs list + new (form: route + rider + date) + [id] — RIDER MOBILE UI similar to production batches: big stop cards with "Mark Delivered" / "Mark Missed" / "Photo placeholder" / Notes per stop. Sticky progress bar at top: X of Y stops complete. Big "Complete Run" button at bottom.
- /delivery/board — today's runs grouped by kitchen with completion %.
