# Customers + Subscription Plans + Subscriptions

Schema:
- `dim_subscription_frequencies` (SMALLINT PK, code UNIQUE, name, deliveries_per_week NUMERIC(4,2), deprecated_at) — UNIVERSAL seed: daily (7), 3x_week (3), 5x_week (5), weekly (1), biweekly (0.5), monthly (0.25), custom (0)
- `fct_customers` (tenant_id, location_id UUID FK nullable to fct_locations, name, slug, email TEXT, phone TEXT, address_jsonb, delivery_notes TEXT, acquisition_source TEXT, status TEXT CHECK in ('prospect','active','paused','churned','blocked') DEFAULT 'active', lifetime_value NUMERIC DEFAULT 0, properties JSONB, audit/ts/soft-delete) — UNIQUE (tenant_id, slug) WHERE deleted_at IS NULL
- `fct_subscription_plans` (tenant_id, name, slug, description, frequency_id FK, price_per_delivery NUMERIC, currency_code CHAR(3), status TEXT CHECK in ('draft','active','archived') DEFAULT 'active', properties JSONB, audit/ts/soft-delete) — UNIQUE (tenant_id, slug) WHERE deleted_at IS NULL
- `dtl_subscription_plan_items` (plan_id FK, product_id FK, variant_id UUID FK nullable to fct_product_variants, qty_per_delivery NUMERIC(8,2) NOT NULL CHECK >0, position INT, notes, audit/ts/soft-delete) — UNIQUE (plan_id, product_id, variant_id) WHERE deleted_at IS NULL
- `fct_subscriptions` (tenant_id, customer_id FK, plan_id FK, service_zone_id UUID FK nullable to fct_service_zones, start_date DATE NOT NULL, end_date DATE NULL, status TEXT CHECK in ('active','paused','cancelled','ended') DEFAULT 'active', paused_from DATE NULL, paused_to DATE NULL, billing_cycle TEXT, properties JSONB, audit/ts/soft-delete)
- `evt_subscription_events` (subscription_id FK, event_type TEXT CHECK in ('started','paused','resumed','cancelled','plan_changed','frequency_changed'), from_date DATE, to_date DATE NULL, reason TEXT, metadata JSONB, ts TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP, performed_by_user_id UUID NOT NULL)

Views: v_customers (join location + computed subscription_count); v_subscription_plans (join frequency_name + item_count); v_subscription_plan_items (join product + variant + computed line_price); v_subscriptions (join customer + plan + service_zone + days_active computed).

Sub-features:
- `80_customers` (5 files) — customers CRUD + /search endpoint with q filter
- `85_subscriptions` (5 files) — subscription_plans CRUD + items nested + subscriptions CRUD + frequencies read + pause/resume via PATCH status; POST /subscriptions/{id}/events for manual event logs (rare; most events auto-emit on PATCH)

Endpoints under `/v1/somaerp/customers/*` + `/v1/somaerp/subscriptions/*`:
- GET /customers (filter status, location_id, q), POST, GET /{id}, PATCH /{id}, DELETE /{id}
- GET /subscriptions/frequencies (read)
- GET /subscriptions/plans (filter status, frequency_id), POST, GET /{id}, PATCH, DELETE
- GET /subscriptions/plans/{id}/items, POST, PATCH /{item_id}, DELETE /{item_id}
- GET /subscriptions (filter customer_id, plan_id, status, start_date range), POST (body: customer_id, plan_id, service_zone_id optional, start_date, billing_cycle, properties), GET /{id}, PATCH (status transitions emit events: active↔paused, active→cancelled, active→ended, paused→cancelled), DELETE
- GET /subscriptions/{id}/events (read event log)

Status transitions emit evt_subscription_events automatically with appropriate event_type.

Frontend:
- /customers landing with stats (count by status)
- /customers list + filters + /customers/new (form: name, email, phone, location, delivery_notes, acquisition_source, status default active, address JSON)
- /customers/[id] detail page with subscriptions list inline (read-only link to /subscriptions/[sub_id])
- /subscriptions landing (2 cards: Plans, Active Subscriptions)
- /subscriptions/plans list + /new (form: name, slug, description, frequency select, price_per_delivery, currency, status) + /[id] (detail + item-add inline form)
- /subscriptions/list + /new (form: customer search, plan select, service_zone optional, start_date default today, billing_cycle) + /[id] (detail + events log + big Pause/Resume/Cancel buttons)
