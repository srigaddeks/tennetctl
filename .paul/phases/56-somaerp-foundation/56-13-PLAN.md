# Reporting Views — cross-layer read-only dashboards

No new tables. Only views + compute endpoints that roll up data from prior layers.

## Views (migration 012)

- `v_dashboard_today` — one row per tenant: today's date, active_batches_count, completed_batches_count, in_transit_runs_count, completed_runs_count, scheduled_deliveries_count, completed_deliveries_count, active_subscriptions_count, total_kitchens_active (computed from today's activity).
- `v_batch_yield_daily` — per (tenant_id, kitchen_id, product_id, run_date): SUM(actual_qty), SUM(planned_qty), yield_pct AVG, batch_count. Used for trends.
- `v_batch_cogs_daily` — per (tenant_id, kitchen_id, product_id, run_date): total_cogs SUM, cogs_per_unit AVG, batch_count. Uses v_batch_summary base.
- `v_inventory_reorder_alerts` — per (tenant_id, kitchen_id, raw_material_id): current_qty, reorder_point_qty extracted via raw_material.properties->>'reorder_point_qty' (nullable — if null, skip); alert_level computed (`critical` if qty<=0, `low` if qty<reorder_point, else `ok`); primary_supplier_name.
- `v_procurement_spend_monthly` — per (tenant_id, year_month, kitchen_id, supplier_id): total_spend, run_count, line_count. Uses fct_procurement_runs + dtl_procurement_lines.
- `v_subscription_revenue_projected` — per (tenant_id, subscription_id): per-month projected revenue = plan.price_per_delivery × frequency.deliveries_per_week × 4.333.
- `v_fssai_compliance_batches` — per batch: batch_id, run_date, product_name, recipe_version, lot_numbers ARRAY (aggregated from consumption lines), qc_results JSONB aggregate, kitchen_name, completed_by_user. Used for CSV export.

## Sub-feature `99_reports` (5 files)

- schemas: DashboardTodayOut, YieldTrendPoint, CogsTrendPoint, InventoryAlertOut, ProcurementSpendPoint, RevenueProjectionOut, ComplianceBatchRow.
- repository: select_dashboard_today, list_yield_trends(from, to, kitchen_id?, product_id?, bucket=daily|weekly|monthly), list_cogs_trends(same filters), list_inventory_alerts(kitchen_id?, severity?), list_procurement_spend(from, to, kitchen_id?, supplier_id?, bucket=monthly), list_revenue_projection(status?=active, as_of_date?), list_compliance_batches(from, to, product_id?, format='json'|'csv')
- service: all read-only. Audit `somaerp.reports.{dashboard_today,yield_trends,cogs_trends,inventory_alerts,procurement_spend,revenue_projection,compliance_export}.queried` (fire-and-forget, not transactional).
- routes: APIRouter prefix=`/v1/somaerp/reports` tags=["reports"]:
  - `GET /dashboard/today` (optional ?date= to query historical)
  - `GET /yield/trends?from=&to=&kitchen_id=&product_id=&bucket=`
  - `GET /cogs/trends?from=&to=&kitchen_id=&product_id=&bucket=`
  - `GET /inventory/alerts?kitchen_id=&severity=`
  - `GET /procurement/spend?from=&to=&kitchen_id=&supplier_id=&bucket=`
  - `GET /revenue/projection?status=&as_of=`
  - `GET /compliance/batches?from=&to=&product_id=&format=` — when format=csv return `Content-Type: text/csv` with CSV body; otherwise JSON envelope

## Frontend

- /reports landing (6 cards: Today's Dashboard, Yield & COGS, Inventory Alerts, Procurement Spend, Revenue Projection, FSSAI Compliance Export)
- /reports/today — KPI tiles + active batches list + active runs list + today's deliveries progress
- /reports/yield — time-series chart (simple bar/line with div widths — no chart library per simplicity rule) for yield + COGS, filters (from/to/kitchen/product/bucket)
- /reports/inventory-alerts — table of critical + low alerts with quick "Start Procurement" link to /procurement/runs/new
- /reports/procurement — monthly spend table + totals
- /reports/revenue — projected revenue per active subscription (listed) + monthly rollup
- /reports/compliance — filter form (from + to + product) + download CSV button (triggers `getComplianceBatches({format: 'csv'})` which fetches and blob-downloads)
