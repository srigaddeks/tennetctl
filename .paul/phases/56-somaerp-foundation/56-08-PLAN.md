<objective>
Quality Control vertical. Schema: dim_qc_check_types (visual/smell/weight/temperature/taste/lot/documentation — UNIVERSAL seed), dim_qc_stages (pre_production/in_production/post_production/fssai — UNIVERSAL seed), dim_qc_checkpoints (TENANT-SPECIFIC — per recipe-step checkpoint templates with criteria_jsonb; NO seed), evt_qc_checks (immutable append-only per-batch check results — NO seed). Views v_qc_checkpoints, v_qc_checks (with joined check_type, stage, result labels).

Sub-features:
- `50_quality` — checkpoints CRUD + checks read + checkpoint-for-recipe association (checkpoints can belong to a recipe step OR a raw_material receiving check). 

Endpoints under `/v1/somaerp/quality/*`:
- GET /check-types (read), GET /stages (read)
- GET /checkpoints (filter by recipe_step_id, raw_material_id, stage_id, status), POST, GET one, PATCH, DELETE (soft)
- GET /checks (filter by batch_id, checkpoint_id, outcome, date range), POST (create immutable result row), GET one; NO PATCH/DELETE (append-only).

Frontend:
- /quality landing (2 cards: Checkpoints, Recent Checks)
- /quality/checkpoints list + new (form: Stage, Check Type, Scope (recipe_step_id OR raw_material_id OR kitchen_id), Name, Criteria JSON)
- /quality/checks list (read-only feed + filters)

**No seed beyond dim_qc_check_types and dim_qc_stages (both universal food-industry concepts).**
</objective>

Same conventions: TIMESTAMP, uuid7, properties JSONB, audit emission `somaerp.quality.{checkpoints,checks}.{created,updated,deleted}`. No external SaaS. pytest + MCP deferred.
