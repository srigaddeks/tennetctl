## Goal

Ship CATALOG vertical end-to-end: product lines + products + product variants (nested) + tag attach/detach. Seed Soma Delights Cold-Pressed Drinks product line + 6 SKUs (Green Morning, Citrus Immunity, Beetroot Recharge, Hydration Cooler, Tropical Detox, Turmeric Ginger Shot) + default variants + wellness tags (immunity/energy/detox/hydration/gut/skin).

## Purpose

Catalog is the dependency for recipes (56-07), production batches (56-10), subscriptions (56-11), AND for kitchen capacity (56-06 needs fct_product_lines FK). Unblocks the rest of phase 56.

## Output

- Migration: 4 tables (dim_product_categories, dim_product_tags, fct_product_lines, fct_products, fct_product_variants) + 1 link table (lnk_product_tags) + 3 views (v_product_lines, v_products, v_product_variants) in schema "11_somaerp"
- 2 seed YAMLs (categories + tags, globally-unique filenames)
- 2 sub-features: `20_product_lines` (lines + categories read) + `25_products` (products + nested variants + tag attach + tags read), 5-file pattern each
- 10+ endpoints under /v1/somaerp/catalog/*
- 5 frontend pages: /catalog landing + /catalog/product-lines list+new + /catalog/products list+new
- Real-DB pytest for both sub-features
- MCP Playwright walk verifying E2E with authed session
