# Plan 42-01 — Flow Schema Definition (DAG, Port Types, Versioning)

**Phase:** 42 — Canvas + Visual Flow Viewer (v0.4.0)
**Goal:** Persist flow definitions — the DAG of node instances and typed edges that the canvas renders. Establish the storage shape, port-type system, and draft → publish → immutable-version workflow so backend (Plan 43-01) and frontend (Plan 44-01) have a stable schema to build against.

---

<objective>

## Goal
Introduce a `flows` sub-feature inside the catalog feature. A **flow** is a DAG of node-instance vertices joined by typed edges (`next | success | failure | true | false`). Each node instance references a stable `node_key` (resolved live against the in-memory node registry — never copied), carries a per-instance `config_json` validated against the node's `config_schema`, and exposes typed input/output **ports** derived from the node's schemas. Flows go through a draft → publish lifecycle (ADR-020): draft flows are mutable, published flow versions are byte-frozen, and the canvas always renders a specific version.

## Purpose
Without a stored DAG schema there is no canvas to render and no execution trace to overlay. Plan 43-01 needs to load `(flow_version_id) -> nodes[] + edges[] + port_types`, and Plan 44-01 needs the same shape over HTTP. Locking this contract in Plan 42 is the orthogonality lever — backend and frontend can be built in parallel against the same Pydantic + view shape.

## Output
- `dim_catalog_flow_status` (SMALLINT PK seeded: draft=1, published=2, archived=3)
- `dim_catalog_flow_edge_kind` (SMALLINT PK seeded: next=1, success=2, failure=3, true_branch=4, false_branch=5)
- `dim_catalog_port_type` (SMALLINT PK seeded: any=1, string=2, number=3, boolean=4, object=5, array=6, uuid=7, datetime=8, binary=9, error=10) — used for typed-port compatibility check
- `fct_catalog_flows` (id UUID v7, org_id, workspace_id, slug TEXT, name TEXT, current_version_id UUID NULL FK, status_id FK, created_at, updated_at, deleted_at) — UNIQUE (org_id, slug) WHERE deleted_at IS NULL; the flow is the stable identity, versions are immutable snapshots
- `fct_catalog_flow_versions` (id UUID v7, flow_id FK, version_number INT, status_id FK, dag_hash CHAR(64), published_at NULL, published_by_user_id NULL FK, created_at, deleted_at NULL) — UNIQUE (flow_id, version_number); only `draft` rows are mutable
- `dtl_catalog_flow_nodes` (id UUID v7, flow_version_id FK, node_key TEXT, instance_label TEXT, config_json JSONB, position_x INT, position_y INT, sort_order SMALLINT) — UNIQUE (flow_version_id, instance_label); `node_key` references live registry, no FK to a `fct_nodes` table (registry is code-first per ADR-018)
- `dtl_catalog_flow_edges` (id UUID v7, flow_version_id FK, from_node_id FK, from_port_key TEXT, to_node_id FK, to_port_key TEXT, edge_kind_id FK, sort_order SMALLINT) — UNIQUE (flow_version_id, from_node_id, from_port_key, to_node_id, to_port_key)
- View `v_catalog_flows` (flow + status_text + current_version_number + node_count + edge_count)
- View `v_catalog_flow_versions` (version + flow_slug + status_text + node_count + published_at + dag_hash)
- `dag.py`: pure `validate_dag(nodes, edges) -> DagValidation` — checks acyclicity (Kahn topo-sort), reachability from entry nodes, every `failure`/`success` edge has a sibling complementary edge, no orphan instance_labels
- `port_resolver.py`: pure `resolve_ports(node_key, registry) -> {inputs: [Port], outputs: [Port]}` and `is_compatible(from_port_type, to_port_type) -> bool` (any matches anything, exact match otherwise; numeric widening allowed: number→any but not boolean→number)
- `version_publish.py`: pure `compute_dag_hash(nodes, edges) -> str` (sha256 of canonical JSON) and `freeze_draft(conn, flow_version_id, user_id)` — atomically marks status=published, sets published_at + published_by_user_id, increments parent flow's `current_version_id`, copies the draft into a brand-new `draft` version_number+1 so editing continues without mutating the published row
- Routes (5 + 2 lifecycle as PATCH bodies, no action endpoints):
  - `GET /v1/flows` (filters: status, q, workspace_id)
  - `POST /v1/flows` (creates flow + initial draft version v1)
  - `GET /v1/flows/{id}` (returns flow + current version summary)
  - `PATCH /v1/flows/{id}` (rename, archive via {status:"archived"}, publish via {publish_version_id:"..."})
  - `DELETE /v1/flows/{id}` (soft-delete)
  - `GET /v1/flows/{id}/versions/{version_id}` (full DAG: nodes[] + edges[] + resolved port types)
  - `PATCH /v1/flows/{id}/versions/{version_id}` (mutate draft only — 409 if status=published)
- ≥20 pytest, no E2E in this plan (UI lives in Plan 44-01)

## AC-1: Flow + draft version creation persists DAG with typed ports
```gherkin
Given a workspace with the node registry containing keys "iam.auth_required", "audit.emit", "core.run_handler"
When operator POSTs /v1/flows with {slug:"public-api", name:"Public API Preset", nodes:[{instance_label:"auth", node_key:"iam.auth_required", config:{scope:"read"}, position:{x:0,y:0}}, {instance_label:"handler", node_key:"core.run_handler", config:{}, position:{x:240,y:0}}, {instance_label:"audit", node_key:"audit.emit", config:{}, position:{x:480,y:0}}], edges:[{from:"auth", from_port:"out", to:"handler", to_port:"in", kind:"success"}, {from:"handler", from_port:"out", to:"audit", to_port:"in", kind:"next"}]}
Then a fct_catalog_flows row is created with status=draft, current_version_id pointing at version_number=1
And dtl_catalog_flow_nodes contains 3 rows with config_json validated against each node's config_schema
And dtl_catalog_flow_edges contains 2 rows with edge_kind_id resolved from "success"/"next"
And GET /v1/flows/{id}/versions/{vid} returns nodes[] each with resolved {inputs:[{key,type}], outputs:[{key,type}]} pulled from the live registry — never stored on the row
```

## AC-2: DAG validation blocks cycles and dangling edges
```gherkin
Given a draft flow version
When operator PATCHes the version with edges that form a cycle a→b→c→a
Then service returns 422 with error.code="DAG_CYCLE" and details listing the cycle path
And no rows are written

Given an edge whose from_node_id has no instance with the given from_port_key
When dag.validate_dag runs
Then it returns DagValidation(ok=false, errors=[{code:"UNKNOWN_PORT", node:"handler", port:"banana"}])
And the PATCH is rejected with 422

Given a "true_branch" edge from a control node with no sibling "false_branch" edge from the same node
Then validate_dag rejects with code="MISSING_BRANCH_PAIR"
```

## AC-3: Port type compatibility enforced at edge insert
```gherkin
Given node "split.json" outputs a port "items" of type=array
And node "string.upper" inputs a port "value" of type=string
When operator adds edge {from:"split.json", from_port:"items", to:"string.upper", to_port:"value"}
Then port_resolver.is_compatible(array, string) returns false
And the PATCH is rejected with 422 error.code="PORT_TYPE_MISMATCH" and details {expected:"string", got:"array"}

Given the same source connected to a target port of type=any
Then is_compatible(array, any) returns true
And the edge persists
```

## AC-4: Publish freezes a version, opens a new draft
```gherkin
Given a flow with current_version_id pointing at version_number=1, status=draft
When operator PATCHes /v1/flows/{id} with {publish_version_id: "<v1-id>"}
Then version 1 status flips to published, published_at + published_by_user_id are set, dag_hash is computed
And a new fct_catalog_flow_versions row version_number=2, status=draft is created with all nodes+edges copied
And current_version_id on the flow now points at version 2
And subsequent PATCH /v1/flows/{id}/versions/{v1-id} returns 409 error.code="VERSION_FROZEN"
And dag_hash is identical for version 1 and the cloned version 2 (until version 2 is mutated)
```

## Task 2 — Pure compute modules
- `dag.py`:
  - `validate_dag(nodes: list[NodeInstance], edges: list[Edge]) -> DagValidation` — Kahn topo-sort for acyclicity, reachability set walk, branch-pair check (`true_branch` always has sibling `false_branch` from same node), unknown-port check
  - `topological_order(nodes, edges) -> list[NodeInstance]` reused by canvas layout in 43-01
- `port_resolver.py`:
  - `resolve_ports(node_key, registry) -> ResolvedPorts` — translates node `input_schema` and `output_schema` JSON Schemas into `[Port(key, type)]` lists; the registry is the same `client.catalog` surface external apps consume
  - `is_compatible(from_type: str, to_type: str) -> bool` — `any` matches anything, exact match otherwise, no implicit numeric widening (boolean→number rejected)
- `version_publish.py`:
  - `compute_dag_hash(nodes, edges) -> str` — sha256 over canonical-sorted JSON of (node_key, instance_label, config_json, position) tuples + (from, from_port, to, to_port, kind) tuples
  - `freeze_draft(conn, flow_id, version_id, user_id)` — single transaction: SELECT FOR UPDATE the version, assert status=draft, flip to published, write hash, clone all dtl_* rows into a new draft version, update flow.current_version_id

All three pass conn-free where compute-only; only `freeze_draft` takes conn.
