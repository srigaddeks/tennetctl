## Goal
Build the Notify variable system: `13_fct_notify_template_variables` schema, CRUD API at `GET|POST|PATCH|DELETE /v1/notify/templates/{template_id}/variables/{var_id}`, safelist enforcement for dynamic SQL (SELECT-only, no DML keywords, allowed param keys), `resolve_variables(conn, template_id, context)` function, and updated `notify.templates.render` node that resolves registered variables before Jinja2 rendering.

## Purpose
Templates use Jinja2 variables. Without a registry, callers must supply every variable at render time. The variable system lets templates pre-register static literals ("brand_name" = "ACME Corp") and dynamic SQL queries ("user_display_name" = SELECT display_name FROM v_users WHERE id = $1). The render node resolves all registered variables automatically, merging with caller-supplied overrides. Plans 11-03+ (subscriptions/delivery) rely on `resolve_variables` to enrich notification payloads from audit event context.

    **safelist.py** (pure module, no DB, no notify imports):
    ```python
    import re

    _FORBIDDEN_RE = re.compile(
        r'\b(INSERT|UPDATE|DELETE|CREATE|DROP|ALTER|TRUNCATE|GRANT|REVOKE|EXECUTE|EXEC|COPY)\b',
        re.IGNORECASE,
    )
    _ALLOWED_CONTEXT_KEYS = frozenset({
        "actor_user_id", "org_id", "workspace_id", "event_metadata"
    })

    def validate_dynamic_sql(sql_template: str, param_bindings: dict | None) -> None:
        """
        Raises ValueError if sql_template fails safelist checks (save-time validation).
        Runtime enforcement: statement_timeout=2s + SET LOCAL within each query.
        """
        stripped = (sql_template or "").strip()
        if not stripped.upper().startswith("SELECT"):
            raise ValueError("dynamic_sql must start with SELECT")
        if _FORBIDDEN_RE.search(stripped):
            raise ValueError("dynamic_sql contains forbidden DML/DDL keyword")
        for param, context_key in (param_bindings or {}).items():
            if context_key not in _ALLOWED_CONTEXT_KEYS:
                raise ValueError(
                    f"param_bindings key {context_key!r} is not allowed. "
                    f"Allowed: {sorted(_ALLOWED_CONTEXT_KEYS)}"
                )
    ```

    - `resolve_variables(conn, *, template_id: str, context: dict) -> dict[str, Any]`:
      ```python
      rows = await conn.fetch(SELECT from v_notify_template_variables WHERE template_id=$1)
      result = {}
      for row in rows:
          if row["var_type"] == "static":
              result[row["name"]] = row["static_value"]
          else:  # dynamic_sql
              bindings = dict(row["param_bindings"] or {})
              # Build positional args: bindings = {"$1": "actor_user_id", "$2": "org_id"}
              args = []
              for i in range(1, len(bindings) + 1):
                  ctx_key = bindings.get(f"${i}")
                  args.append(context.get(ctx_key) if ctx_key else None)
              # Set 2-second timeout, reset to 0 after query
              await conn.execute("SET LOCAL statement_timeout = '2000'")
              try:
                  dyn_row = await conn.fetchrow(row["sql_template"], *args)
                  result[row["name"]] = dyn_row[0] if dyn_row else None
              finally:
                  await conn.execute("SET LOCAL statement_timeout = '0'")
      return result
      ```
