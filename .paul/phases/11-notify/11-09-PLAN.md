<objective>
## Goal
Ship the Template Designer UI: a full-stack vertical delivering template list, designer page (body editor + live preview + variable picker + test-send), and enabling campaign creation now that templates are browsable.

## Purpose
Templates are the core content atom in Notify — subscriptions, campaigns, and the transactional API all reference them. The designer unblocks campaign creation (disabled since 11-08) and is the last piece before the pure API (11-10) and analytics (11-11).

## Output
- `POST /v1/notify/templates/{id}/test-send` backend route
- `/notify/templates` list page with create-template dialog
- `/notify/templates/[id]` designer: metadata form + email body editor + live HTML preview + variable picker + test-send dialog
- Campaign create dialog unblocked with template picker
- Robot E2E: create → design → preview → test-send
</objective>
