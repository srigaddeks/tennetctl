# Plan 11-11: Delivery Analytics UI + Notify Explorer

**Phase:** 11 (Notify)
**Depends on:** 11-10
**Autonomous:** true

## Objective

Add a delivery explorer page and campaign analytics detail view to complete the Notify frontend surface.

## Acceptance Criteria

- [ ] `GET /v1/notify/campaigns/{id}/stats` returns delivery counts per status
- [ ] `/notify/deliveries` page lists deliveries with org/status/channel filters
- [ ] `/notify/campaigns/[id]` page shows campaign stats (sent/delivered/opened/clicked/bounced)
- [ ] `NotifyDelivery`, `NotifyCampaignStats` types in api.ts
- [ ] `use-deliveries.ts` hook with `useDeliveries` and `useCampaignStats`
- [ ] Deliveries nav entry added to features.ts
- [ ] All existing tests still pass

## Tasks

### Task 1: Backend — campaign stats endpoint

**Action:**
- Add `list_campaign_deliveries_stats` to `sub_features/06_deliveries/repository.py` — counts deliveries by status for a campaign
- Add `GET /v1/notify/campaigns/{id}/stats` to `sub_features/10_campaigns/routes.py` (delegates to deliveries service)

**Verify:** `pytest tests/test_notify_campaigns_api.py -q` passes

### Task 2: Frontend types + hooks

**Action:**
- Add `NotifyDelivery`, `NotifyCampaignStats` to `frontend/src/types/api.ts`
- Create `frontend/src/features/notify/hooks/use-deliveries.ts` with:
  - `useDeliveries(orgId, filters)` — GET /v1/notify/deliveries
  - `useCampaignStats(campaignId)` — GET /v1/notify/campaigns/{id}/stats
- Add `Deliveries` nav entry to `frontend/src/config/features.ts`

### Task 3: Frontend pages

**Action:**
- Create `frontend/src/app/(dashboard)/notify/deliveries/page.tsx` — Delivery explorer:
  - Filter bar: status, channel dropdowns
  - Table: recipient, template, channel, status, created_at
  - Uses `useMe()` for orgId
- Create `frontend/src/app/(dashboard)/notify/campaigns/[id]/page.tsx` — Campaign detail:
  - Header: campaign name, status badge, scheduled_at
  - Stats row: sent / delivered / opened / clicked / bounced / failed counts with % rates
  - Back link to /notify/campaigns

**Verify:** TypeScript build clean (`npm run build` or `npx tsc --noEmit`)
