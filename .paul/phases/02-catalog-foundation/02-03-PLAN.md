<objective>
## Goal
Ship the node runner — the `run_node(key, ctx, inputs) -> dict` dispatch mechanism — with every execution primitive from NCP v1 §6–§9: `NodeContext` (audit scope + tracing), `Node` base class (contract for all node handlers), execution policy enforcement (timeout, retries on `TransientError`, tx modes caller/own/none), and pluggable authorization hook. Runner becomes the one and only way sub-features call each other.

## Purpose
Phase 3 (IAM + Audit) needs `run_node` to call `audit.emit` from every IAM effect node. Phase 4+ feature verticals will use `run_node` for every cross-sub-feature call. Without the runner, the catalog DB is a pretty index with nothing to dispatch against. After this plan, the full NCP v1 pipeline is operational: manifest → catalog DB → `run_node` → Python handler → return.

## Output
- `backend/01_catalog/context.py` — `NodeContext` frozen dataclass
- `backend/01_catalog/node.py` — `Node` base class (key, kind, Input, Output, run)
- `backend/01_catalog/errors.py` — runtime error hierarchy (`TransientError`, `DomainError`, `NodeTimeout`, `NodeAuthDenied`, `NodeNotFound`, `NodeTombstoned`, `IdempotencyRequired`)
- `backend/01_catalog/authz.py` — `check_call(ctx, node_meta)` + `register_checker(fn)` extension point
- `backend/01_catalog/runner.py` — `run_node(pool_or_conn, key, ctx, inputs)` enforcing execution policy
- Public exports added to `backend/01_catalog/__init__.py`
- Three additional fixture nodes (echo/slow/flaky) exercising the runner paths
- `tests/test_catalog_runner.py` with 6+ tests covering: happy path, timeout, retries on TransientError, auth deny, unknown key, tombstoned node

## AC-1: NodeContext propagates audit + tracing fields
```gherkin
Given a NodeContext with user_id="u1", session_id="s1", org_id="o1", workspace_id="w1", trace_id="t1", span_id="sp1"
When run_node is called with this ctx
Then the downstream handler receives a child NodeContext whose trace_id == parent.trace_id
And the child's parent_span_id == parent.span_id
And the child's span_id is a fresh UUID v7 distinct from parent.span_id
And user_id/session_id/org_id/workspace_id/request_id/audit_category propagate unchanged
```

## AC-2: Happy path — run_node executes handler and returns validated output
```gherkin
Given the catalog contains the fixture node "core.sample.echo" (control, tx=none, handler returns Input echoed)
When run_node(pool, "core.sample.echo", ctx, {"msg": "hi"}) is called
Then the handler is resolved via the catalog's handler_path
And inputs are validated against the handler's Input Pydantic class
And the handler's output dict is validated against Output
And the returned dict matches {"msg": "hi", "echoed": true}
```

## AC-3: Execution policy — timeout cancels slow nodes
```gherkin
Given a fixture node "core.sample.slow" with timeout_ms=200 that sleeps 2 seconds
When run_node is called with default ctx
Then the call raises NodeTimeout (CAT_TIMEOUT)
And it raises within ~250ms (not after the full 2s sleep)
```

## AC-4: Retries fire only on TransientError
```gherkin
Given a fixture node "core.sample.flaky" with retries=2 configured in the manifest
And the node's handler raises TransientError on calls #1 and #2, returns {"ok": true} on call #3
When run_node is called with an idempotency_key in inputs
Then the handler is invoked 3 times total
And the final return value is {"ok": true}

Given the same fixture node without an idempotency_key in inputs
When run_node is called
Then IdempotencyRequired (CAT_IDEMPOTENCY_REQUIRED) is raised before any handler invocation

Given a fixture node that raises DomainError (not transient)
When run_node is called with retries=2
Then the handler is invoked exactly once (no retries on domain errors)
And DomainError propagates to the caller unchanged
```

## AC-5: Authorization hook blocks or allows
```gherkin
Given the default authz checker is active
And ctx.audit_category == "system"
When run_node is called against any non-tombstoned node
Then the call proceeds (system callers are always allowed)

Given ctx.audit_category == "user" and ctx.user_id is None
When run_node is called
Then NodeAuthDenied (CAT_AUTH_DENIED) is raised before handler invocation

Given a custom checker is registered via authz.register_checker that rejects calls to keys matching "core.sample.*"
When run_node is called with key "core.sample.echo"
Then NodeAuthDenied is raised
And the custom checker runs before the default checker
```

## AC-6: Unknown / tombstoned keys fail cleanly
```gherkin
Given the catalog does not contain key "core.sample.nonexistent"
When run_node is called with that key
Then NodeNotFound (CAT_NODE_NOT_FOUND) is raised

Given the catalog contains key "core.sample.ping" with tombstoned_at set to NOW()
When run_node is called with that key
Then NodeTombstoned (CAT_NODE_TOMBSTONED) is raised
```

## AC-7: Transaction modes respected
```gherkin
Given a node with tx_mode="none"
When run_node is called
Then the handler receives ctx.conn is None (no DB connection attached)

Given a node with tx_mode="own"
When run_node is called without a caller conn
Then the runner acquires a fresh connection from the pool
And wraps the handler call in a BEGIN/COMMIT transaction
And on handler exception rolls back

Given a node with tx_mode="caller" and ctx.conn is already set to a caller's conn
When run_node is called
Then the handler receives the same caller conn (no new acquisition)
```

    Checker = Callable[[NodeContext, dict], Awaitable[None]]  # raises NodeAuthDenied to reject

    _checkers: list[Checker] = []

    def register_checker(fn: Checker) -> None: _checkers.append(fn)

    async def check_call(ctx: NodeContext, node_meta: dict) -> None:

    **`backend/01_catalog/runner.py`** — the dispatch mechanism.

    Top-level function signature:
    ```python
    async def run_node(
        pool: asyncpg.Pool,
        key: str,
        ctx: NodeContext | None = None,
        inputs: dict | None = None,
    ) -> dict:
    ```

    Behavior — follow NCP §7 + §8 in order:

    1. **Normalize args:** ctx = ctx or NodeContext.system(); inputs = inputs or {}.

    2. **Catalog lookup:**
       ```
       SELECT n.id, n.key, n.handler_path, n.version, n.emits_audit,
              n.timeout_ms, n.retries, n.deprecated_at, n.tombstoned_at,
              k.code AS kind_code, t.code AS tx_mode_code
       FROM "01_catalog"."12_fct_nodes" n
         JOIN "01_catalog"."02_dim_node_kinds" k ON k.id = n.kind_id
         JOIN "01_catalog"."03_dim_tx_modes" t ON t.id = n.tx_mode_id
       WHERE n.key = $1
       ```
       - No row → raise NodeNotFound(key).
       - tombstoned_at IS NOT NULL → raise NodeTombstoned(key).
       - deprecated_at IS NOT NULL → `logger.warning("Calling deprecated node %r", key)` but continue.

    3. **Authorization:** build node_meta dict from the row; `await authz.check_call(ctx, node_meta)`.

    4. **Resolve handler:** import the handler class via the same `_handler_import_path` logic as the loader (prefer extracting it to a shared helper `backend/01_catalog/_handler_resolver.py` OR leave it as a private function in runner.py that duplicates the 10 lines — extraction is cleaner, pick that).

    5. **Validate inputs:** `validated = handler_cls.Input(**inputs)`. Pydantic ValidationError → wrap as DomainError (not retryable).

    6. **Idempotency check:** if `meta.retries > 0` and `"idempotency_key" not in inputs`:
       - Raise IdempotencyRequired(node_key=key).
       - (A full idempotency cache is out of scope for v1 — deferred to v0.1.5; for now we REQUIRE the key but don't dedupe by it.)

    7. **Child context:** `child_ctx = ctx.child_span(key)`.

    8. **Transaction mode:** acquire `conn` per meta.tx_mode_code:
       - "none": child_ctx with conn=None.
       - "caller": if ctx.conn is set, reuse it; else acquire from pool and reuse for the whole call (do NOT open an inner tx; this mode means "join caller's tx", and if there's no caller tx we just take a conn).
       - "own": acquire a fresh conn and open a transaction; rollback on any exception from the handler, commit otherwise.
       - Use `dataclasses.replace(child_ctx, conn=the_conn)` to attach.

    9. **Timeout wrapping:**
       - effective_ms = ctx.timeout_override_ms if set else meta.timeout_ms.
       - Wrap the handler call in `asyncio.wait_for(coro, timeout=effective_ms/1000.0)`.
       - On `asyncio.TimeoutError`, raise `NodeTimeout(key)`.

    10. **Retry loop:** if meta.retries > 0:
        - Call the handler up to (retries + 1) times.
        - Retry ONLY on subclasses of `TransientError` (including user subclasses).
        - Do NOT retry on DomainError, NodeTimeout, NodeAuthDenied, ValidationError, or any non-RunnerError.
        - Small linear backoff: `await asyncio.sleep(0.05 * attempt)` between attempts.
        - If final attempt still raises, propagate the last exception.

    11. **Output validation:**
        - The handler returns either a dict or a BaseModel.
        - If dict: construct `handler_cls.Output(**result)` to validate.
        - If BaseModel: call `.model_dump()` after verifying isinstance(result, handler_cls.Output).
        - Return the validated dict.

    12. **Effect audit check (runtime safety net):**
        - If meta.kind_code == "effect" and meta.emits_audit is False:
          - Raise DomainError ("effect node must emit audit" — this should never happen because the DB CHECK constraint prevents it, but defense in depth).

    Keep the runner ~150 lines. Don't over-engineer — v0.1.5 adds idempotency dedupe, NATS async dispatch, and gateway sync.

    **`backend/01_catalog/__init__.py`** — add public exports:
    - `run_node`, `NodeContext`, `Node`, `register_checker`, `clear_checkers`
    - Error classes: `RunnerError`, `NodeNotFound`, `NodeTombstoned`, `NodeAuthDenied`, `NodeTimeout`, `IdempotencyRequired`, `TransientError`, `DomainError`
    - Keep existing exports intact.

    Avoid:
    - Blocking I/O inside the runner (everything async)
    - Silent failures — every exception path should surface a specific error class with code
    - Running authz after handler — must run before
    - Swallowing the last exception in the retry loop (propagate it)
    - Introducing a new dependency — stdlib asyncio + asyncpg + existing modules suffice
