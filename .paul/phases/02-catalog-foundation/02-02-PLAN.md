<objective>
## Goal
Implement the Python `backend/01_catalog/` module: Pydantic manifest parser (NCP v1 §3 grammar), boot upsert into `01_catalog` fct tables (§11 sequence), cross-import linter (§10 rule), and the `/tnt` Claude Code skill. Prove the pipeline end-to-end with a fixture feature that registers a throwaway node.

## Purpose
Without the loader, every manifest is just a document — the catalog DB stays empty and `run_node` (Plan 02-03) has nothing to dispatch against. This plan makes the catalog *live*: when the backend starts, any `feature.manifest.yaml` under `backend/02_features/` is parsed, validated, and upserted into `fct_features / fct_sub_features / fct_nodes`. The linter makes the cross-import rule *enforceable* rather than aspirational. The `/tnt` skill gives every coding agent a single 40-line primer on the whole system.

## Output
- `backend/01_catalog/` Python package (6 files incl. `__init__.py` + CLI entrypoint)
- Boot loader wired into `backend/main.py` lifespan (after pool, before yield)
- Cross-import linter runnable as `python -m backend.01_catalog.cli lint`
- `.claude/skills/tnt.md` — single skill, ≤50 lines, explains the pattern
- Test fixture feature (`99_test_fixture`) living under `tests/fixtures/features/`
- Smoke test (`tests/test_catalog_loader.py`) that boots the loader against the fixture and asserts catalog rows
</objective>

      - `FeatureManifest`: apiVersion (Literal["tennetctl/v1"]), kind (Literal["Feature"]), metadata, spec.depends_on_modules (list[str] default []), spec.sub_features (list[SubFeatureManifest])
      - Model validator on FeatureManifest: every sub_feature.key must start with `metadata.key + "."`
      - Function `parse_manifest(path: Path) -> FeatureManifest`: reads YAML via `yaml.safe_load`, constructs `FeatureManifest(**data)` — Pydantic ValidationError wrapped as ManifestInvalid
      - Function `discover_manifests(root: Path) -> list[Path]`: returns all `feature.manifest.yaml` under `{root}/backend/02_features/*/` and `{root}/tests/fixtures/features/*/` (fixture dir only included when `include_fixtures=True`)

    **`backend/01_catalog/repository.py`** — raw asyncpg upserts, one function per table:
      - `async def upsert_feature(conn, *, key, number, module_id) -> int` — returns id. Uses `INSERT ... ON CONFLICT (key) DO UPDATE SET updated_at = CURRENT_TIMESTAMP RETURNING id`. Number + module_id uniqueness enforced by DB constraints; FK violation → propagate.
      - `async def upsert_sub_feature(conn, *, key, feature_id, number) -> int`
      - `async def upsert_node(conn, *, key, sub_feature_id, kind_id, handler_path, version, emits_audit, timeout_ms, retries, tx_mode_id) -> int`
      - `async def mark_deprecated(conn, entity_type: str, keys_still_present: set[str]) -> int` — sets `deprecated_at = CURRENT_TIMESTAMP WHERE deprecated_at IS NULL AND key NOT IN (keys_still_present)` for fct_features/fct_sub_features/fct_nodes depending on entity_type.
      - Module / kind / tx_mode code → id lookup via SELECT from dim tables (cached in a module-level dict on first boot).
      - All functions take `conn: asyncpg.Connection` (never pool). Follow `.claude/rules/python.md`.

    **`backend/01_catalog/loader.py`** — orchestrates NCP §11 boot sequence:
      - `async def upsert_all(pool, enabled_modules: frozenset[str], *, fixtures: bool = False) -> LoaderReport`
      - Flow:
         1. discover_manifests(project_root, include_fixtures=fixtures)
         2. For each manifest: parse_manifest → collect (valid) or log+skip (invalid in prod)
         3. Filter: drop features whose metadata.module not in enabled_modules
         4. Resolve handlers: for every node, `import_module(f"backend.02_features.{nn}_{feature_key}.{handler_path}")` must succeed → raise HandlerUnresolved on failure; verify imported object has `key` and `kind` attributes matching manifest (CAT_HANDLER_CONTRACT_MISMATCH) — for fixtures path the feature dir is under tests/fixtures/features so adjust import prefix accordingly
         5. Topsort features by depends_on_modules (use simple Kahn's; cycle → CatalogError)
         6. For each feature, acquire one conn from pool, open tx, upsert feature → sub_features → nodes. Commit per feature.
         7. Deprecation sweep: call mark_deprecated with the set of keys observed.
      - Returns `LoaderReport` dataclass: features_upserted, sub_features_upserted, nodes_upserted, deprecated, errors (list of (manifest_path, error_code, message)).
      - Strict mode (v1): any single error fails the whole boot. Log all errors first via `logger.error`, then raise the first one after the sweep. (Tolerant mode deferred to v0.1.5.)

    **`backend/main.py`** — add catalog upsert to lifespan:
      - Between `pool = await _db.create_pool(...)` and `yield`, call:
        ```python
        _catalog = import_module("backend.01_catalog")
        report = await _catalog.upsert_all(pool, config.modules)
        logger.info("Catalog upsert: %d features, %d sub-features, %d nodes",
                    report.features_upserted, report.sub_features_upserted, report.nodes_upserted)
        ```
      - On `CatalogError` raised: re-raise. Lifespan failure = uvicorn exit = desired fail-fast.
      - Do not add a new dependency — `yaml`, `pydantic`, `asyncpg` already present.

    - JSON schema file on disk — Pydantic's derived schema is authoritative
    - Modifying fct_nodes CHECK constraint — validate at parser level *and* DB level; both layers agree

      - Function `check_file(path: Path) -> list[Violation]`:

         - Extract the owning (feature, sub) tuple from the path
         - Parse file with `ast.parse(path.read_text())`
         - Walk every `ast.ImportFrom` node; reconstruct the absolute dotted module (handle `level=0` only — relative imports are forbidden project-wide per rules/python.md, and if seen they are a separate violation but out of scope for THIS linter)
         - Match the dotted module against pattern: `backend.02_features.{other_feature}.sub_features.{other_sub}.*`
         - Allowed if: `(other_feature, other_sub) == (feature, sub)` (same sub-feature) OR the remainder starts with `nodes.` (node imports permitted per NCP §10 exception) OR the module is `backend.01_core.*` / `backend.01_catalog.*`
