## Design confirmed (prior discussion)
- Feature number: 9. Module + schema: `featureflags` / `"09_featureflags"`.
- Sub-features (5): flags, permissions, rules, overrides, evaluations. All empty shells in this plan.
- Scopes: global / org / application (workspace deferred). fct_flags gets `scope_id` + nullable `org_id` + nullable `application_id` in 09-02.
- Environments: dev / staging / prod / test, seeded at bootstrap.
- Permissions: per-flag role grants via `lnk_role_flag_permissions` (built in 09-03).
- Permission hierarchy (rank): view=1, toggle=2, write=3, admin=4.
- `always_on: true` for featureflags feature — same posture as audit (foundational capability; deployments can still no-op it by not creating any flags).
</context>
