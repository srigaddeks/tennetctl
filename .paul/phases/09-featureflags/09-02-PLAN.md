<objective>
## Goal
Ship the `featureflags.flags` sub-feature backend: flag definitions (global / org / application scoped) with per-environment state. Auto-create one `fct_flag_states` row per environment on flag create so toggles are always a PATCH, never a POST.

## Output
- Migration: `10_fct_flags` + `11_fct_flag_states` + `v_flags` view
- Two new Python tables owned by iam.flags sub-feature
- 5 HTTP endpoints under `/v1/flags` + 3 under `/v1/flag-states` (list/get/patch; auto-provisioned on flag create, auto-cascaded on delete)
- 2 catalog nodes: `featureflags.flags.create` (effect, audit) + `featureflags.flags.get` (control, read)
- 6 pytest integration tests
</objective>

## Scope precedence (from design)
- global  → (scope_id=1, org_id=NULL, application_id=NULL)
- org     → (scope_id=2, org_id set, application_id=NULL)
- app     → (scope_id=3, org_id set, application_id set)
- CHECK constraint enforces the combo; three partial unique indexes on flag_key per scope.

## Per-environment state auto-provisioning
- On flag create: insert 4 rows into fct_flag_states (one per seeded env).
- Default `is_enabled = false` in all envs.
- `env_default_value_jsonb = NULL` means "use flag.default_value".
- PATCH a state to toggle enabled/disabled or override the env default.

## Permission gate for create (v0.1 pragmatic)
- Routes keep `audit_category='setup'` (matches the rest of the backend) — the scope-check helper no-ops for setup.
- Helper is plumbed so when Phase 7 flips to `audit_category='user'`, enforcement activates without a code change in featureflags.
- Minimum required scope (documented, not yet runtime-enforced):
  - global flag → `flags:admin:all`
  - org flag → `flags:admin:org` for that org, OR `flags:admin:all`
  - app flag → `flags:admin:org` for parent org, OR `flags:admin:all`
