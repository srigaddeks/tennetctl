<objective>
## Goal
Ship the admin UI at `/iam/security/policy` that reads every `iam.policy.*` key through the 20-01 backend and lets operators edit global defaults + per-org overrides via existing `PATCH /v1/vault/configs/{id}` endpoints. No new backend routes — UI is a pure consumer of vault.configs + AuthPolicy.

## Purpose
Operators need a dedicated surface to tune auth policy without hand-editing vault rows. Per-org override section makes it clear which values are inherited from global vs explicitly overridden for an org. Immediate invalidation from 20-01 means edits take effect on the next request — no restart.

## Output
- `/iam/security/policy` page with 6 sections (Password, Lockout, Session, Magic Link, OTP, Password Reset) matching the frozen dataclasses from 20-01.
- Global form pre-filled from `GET /v1/vault/configs?scope=global&key_prefix=iam.policy.`; Save calls PATCH per dirty field.
- Per-org override section: list orgs with any `iam.policy.orgs.{org_id}.*` rows; row-expand shows overrides with "Remove override" (DELETE) and "Add override" (POST).
- Robot E2E: load page → edit password.min_length → save → observe toast → reload → value persists; add org override → observe row → delete override.
</objective>
