<objective>
## Goal
Close the IAM audit-coverage gaps from the pre-OSS audit and make password reset revoke all active sessions for the user atomically. Every auth-adjacent state change now emits an audit event. No new sub-features; only additive emissions + one session-revocation call.

## Purpose
The audit gap list from the pre-OSS audit: OTP verify (email + TOTP), TOTP enrollment/deletion, magic-link failed-consume, password-reset request/complete, credentials verify-failed. Without these, security forensics and Notify `critical` alerts can't fire. Password reset also needs to invalidate other sessions — standard industry expectation.
</objective>
