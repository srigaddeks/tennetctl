# Plan 12-01: Magic Link Sub-Feature

**Phase:** 12 (IAM Security Completion)
**Depends on:** Phase 11 (Notify — for email delivery)
**Autonomous:** true

## Objective

Implement magic-link email sign-in: signed single-use tokens, 10-min TTL, rate-limited, consumed-on-use. Integrates with `notify.send.transactional` for email delivery.

## Acceptance Criteria

- [ ] DB: `fct_iam_magic_link_tokens` table with HMAC token hash, TTL, consumed_at
- [ ] `POST /v1/auth/magic-link/request` — create token + enqueue email delivery
- [ ] `POST /v1/auth/magic-link/consume` — validate token + return session (same shape as signin)
- [ ] Token is HMAC-SHA256 signed with vault key `iam.magic_link_signing_key`
- [ ] Rate limit: 3 requests per email per 15 minutes (DB-enforced)
- [ ] Frontend: magic-link tab on signin page
- [ ] pytest: request + consume + rate-limit + expiry tests
- [ ] All existing tests still pass

## Tasks

### Task 1: DB migration — magic link tokens table

**File:** `03_docs/features/03_iam/05_sub_features/11_magic_link/09_sql_migrations/02_in_progress/20260417_029_iam-magic-link.sql`

Table: `"03_iam"."19_fct_iam_magic_link_tokens"` (number 19 = still in fct range)
- id VARCHAR(36) PK (UUID v7)
- user_id VARCHAR(36) NOT NULL FK → fct_users
- email TEXT NOT NULL
- token_hash TEXT NOT NULL UNIQUE (HMAC-SHA256 hex)
- expires_at TIMESTAMP NOT NULL
- consumed_at TIMESTAMP NULL
- ip_address TEXT NULL
- created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

Index: (email, created_at) for rate-limit query
