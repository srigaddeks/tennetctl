# Plan 12-02: OTP Sub-Feature (Email OTP + TOTP)

**Phase:** 12 (IAM Security Completion)
**Depends on:** 12-01
**Autonomous:** true

## Objective

Two OTP methods:
1. **Email OTP** — 6-digit code, 5-min TTL, max 3 attempts, sends via notify
2. **TOTP** — RFC 6238 authenticator app (pyotp), 30s window, setup with QR code

## Acceptance Criteria

- [ ] DB: `fct_iam_otp_codes` (email OTP) + `fct_iam_totp_credentials` (TOTP, multiple per user)
- [ ] Email OTP: `POST /v1/auth/otp/request` + `POST /v1/auth/otp/verify`
- [ ] TOTP: `POST /v1/auth/totp/setup` (returns otpauth URI) + `POST /v1/auth/totp/verify`
- [ ] TOTP: `GET /v1/auth/totp` (list enrolled devices) + `DELETE /v1/auth/totp/{id}`
- [ ] pytest: email OTP request+verify, invalid code, expired code, TOTP setup+verify
- [ ] TypeScript: `useMagicLinkRequest` already done; add `useOtpRequest`, `useOtpVerify`, `useTotpSetup`, `useTotpVerify`
- [ ] Frontend: OTP tab on signin page (email OTP flow) + TOTP setup flow on /auth/security page
- [ ] All existing tests still pass
