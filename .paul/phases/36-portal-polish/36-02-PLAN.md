<objective>
## Goal
Ship the `/notify/suppressions` admin page — closes a 🟡 gap with operator value: suppressed email addresses (hard bounces, complaints, manual holds) are currently invisible in the UI.

## Purpose
Backend endpoints `GET/POST/DELETE /v1/notify/suppressions` have been live since Phase 19 but the UI never landed. Operators have no way to inspect or revive a bounced address without hitting the DB.

## Output
- `NotifySuppressionRow` + `NotifySuppressionAdd` types
- `use-suppressions.ts` hook module (list, add, remove)
- `/notify/suppressions` page with list + add modal + remove confirm
- Nav entry under Notify feature
</objective>
