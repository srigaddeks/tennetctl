<objective>
## Goal
Implement RFC 8058-compliant one-click unsubscribe and an auto-skip suppression list so bounced / unsubscribed recipients don't keep getting mail. Gmail/Yahoo 2024 bulk-sender rules require this.

## Purpose
Without `List-Unsubscribe` + one-click handling, any volume of email gets bulk-filtered. Without a suppression list, hard bounces keep retrying and damage sender reputation.

## Output
- New `16_suppression` sub-feature (full 5-file pattern).
- Email sender injects `List-Unsubscribe` (mailto + URL) and `List-Unsubscribe-Post: List-Unsubscribe=One-Click` headers.
- Signed unsubscribe URL (`/v1/notify/unsubscribe?token=...`) — cookie-less; HMAC via vault.
- Bounce webhook adds to suppression list.
- Email sender + transactional Send skip recipients in suppression list.
</objective>

<acceptance_criteria>

## AC-1: Suppression blocks subsequent sends
```gherkin
Given an email is in `17_fct_notify_suppressions` with reason_code='hard_bounce'
When the email sender picks a delivery for that recipient address
Then the delivery is marked status='unsubscribed' without an SMTP send attempt
```

## AC-2: One-click unsubscribe
```gherkin
Given an email with List-Unsubscribe headers
When the receiving client POSTs to the unsubscribe URL with a valid token
Then the user's preferences row is set to is_opted_in=false for the relevant (category, email channel)
  And the address is inserted into the suppression list (reason_code='manual')
  And the response is 200 with a simple HTML confirmation
```

## AC-3: Email sender injects both List-Unsubscribe variants
```gherkin
Given an outbound email
When the sender builds the MIME message
Then the message has "List-Unsubscribe: <mailto:unsubscribe@...>, <https://host/v1/notify/unsubscribe?token=...>"
  And "List-Unsubscribe-Post: List-Unsubscribe=One-Click"
```

## AC-4: Bounce webhook suppresses
```gherkin
Given POST /v1/notify/email/webhooks/bounce with {delivery_id, reason}
When the webhook handler records the bounce
Then the recipient email is inserted into suppression list (reason_code='hard_bounce')
  And the delivery status becomes 'bounced'
```
