<objective>
## Goal
On a fresh deployment (zero users in fct_users), backend enters "setup mode" where every route except `GET /v1/setup/status` and `POST /v1/setup/initial-admin` returns 503 with a hint pointing to `/setup`. First call creates the super-admin with TOTP mandatory + sets a bootstrap flag `system_initialized=true` in vault.configs. Frontend detects setup mode and renders the admin-creation wizard.

## Purpose
Zero hardcoded default credentials. Clear OSS first-run experience. Forces MFA on the root account (can't be undone without direct DB access).
</objective>

<acceptance_criteria>

## AC-1: Setup-mode gate
```gherkin
Given fct_users is empty AND vault.configs has no system_initialized=true
When any request comes in (except setup routes + health)
Then the SetupModeMiddleware returns 503 with body {ok:false, error:{code:"SETUP_REQUIRED", hint:"/setup"}}
When fct_users has ≥1 user OR system_initialized=true
Then the middleware passes through (setup mode off)
```

## AC-2: Initial admin creation
```gherkin
Given setup mode is active
When POST /v1/setup/initial-admin is called with {email, password, display_name}
Then a super-admin user is created with the seeded "system_admin" role
And a TOTP secret is generated + returned ONCE (user must enroll before finishing)
And 10 backup codes are returned ONCE
And vault.configs sets system_initialized=true (atomic with user creation)
And a session is returned
And audit iam.setup.completed emitted
```

## AC-3: Frontend setup wizard
```gherkin
Given the frontend detects /v1/setup/status returns {setup_required: true}
When any route is visited
Then router middleware redirects to /setup
When /setup renders
Then it shows form (email, password, display_name) → submit shows TOTP QR + backup codes (one-time) → "I saved these" checkbox → continue lands at /
```

## AC-4: Idempotent (no accidental re-init)
```gherkin
Given setup is already complete
When POST /v1/setup/initial-admin is called again
Then it returns 409 with ok=false
```

## AC-5: Tests + E2E
```gherkin
Given fresh DB
When test_setup_mode.py + 23_first_run_setup.robot run
Then setup mode gate + init + second-call-rejection + subsequent normal ops all pass
```
