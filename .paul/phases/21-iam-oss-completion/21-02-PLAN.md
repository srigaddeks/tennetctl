<objective>
## Goal
Ship an admin-driven invite flow: admin invites email + pre-assigned org + roles → Notify sends invite link → recipient lands on accept-invite page → creates account with password (email pre-filled and pre-verified) → auto-assigned to org with roles.

## Purpose
Closed-community OSS deployments need invite-only onboarding (security). Even open deployments want admin invites for team setup. Pre-verified email via invite token (ownership proven by receiving the email).

## Output
- Sub-feature `17_invites/` with CRUD + consume.
- `fct_user_invites` + `dtl_invite_roles` tables.
- Notify template `iam.invite.email`.
- `/iam/invites` admin page + `/auth/accept-invite?token=...` public page.
- Policy keys: `invite.ttl_seconds` (default 604800 = 7 days), `invite.resend_cooldown_seconds` (default 300).
</objective>

## AC-3: Consume creates user + assigns roles + pre-verifies
```gherkin
Given a valid unconsumed unrevoked non-expired invite token

And user is added to invite.org_id (lnk_user_org)
And all invite roles are applied (lnk_user_role)
And the invite is marked consumed_at
And a session is returned (user is auto-signed-in)
And audit iam.invites.consumed emitted
```
