<objective>
## Goal
Ship the Vault backend end-to-end: `02_vault` schema with envelope-encrypted secrets, crypto module (AES-256-GCM with per-secret DEKs wrapped by a root key), `VaultClient` app-singleton with 60s in-memory cache, 5 FastAPI routes under `/v1/vault`, four nodes registered in the catalog (`vault.secrets.get / put / rotate / delete`), boot-time bootstrap of required internal secrets, and pytest integration + crypto tests. Config loader refuses to boot if any `TENNETCTL_*` env var looks like a secret other than the five allowed.

## Purpose
Every subsequent feature (auth in phase 8, OAuth client secrets, signing keys, SMTP creds, third-party API keys) reads its secrets through `VaultClient.get(key)`. This plan is the one-time foundation: envelope encryption, audit-emitting CRUD, backend singleton with SWR cache, and the env-var contract. No UI yet — that is plan 07-02. Auth (phase 8) starts only after 07-01 + 07-02 are closed.

## Output
- ADR-028 recording the envelope-encryption + single-root-key decision
- `02_vault` schema with `10_fct_vault_entries` table and `v_vault_entries` view

- Four catalog nodes wired via `feature.manifest.yaml`
- `backend/main.py` lifespan: instantiate `VaultClient`, run bootstrap, mount router
- `backend/01_core/config.py` enforces env-var contract at startup
- Integration tests: full CRUD via ASGI client + audit emission assertions; crypto unit tests for round-trip, tamper detection, wrong root key failure

## Crypto invariants
- **Root key**: `TENNETCTL_VAULT_ROOT_KEY` = 32 raw bytes base64-encoded (44 chars). Validated on startup.
- **Algorithm**: AES-256-GCM everywhere. `cryptography` library (PyCA), no rolling of our own.
- **Envelope**: every secret has its own randomly-generated 32-byte DEK. DEK encrypts the plaintext with a fresh 12-byte nonce. Root key encrypts the DEK with its own fresh 12-byte nonce. `wrapped_dek` column stores `wrap_nonce || AESGCM(root_key).encrypt(wrap_nonce, dek)` concatenated.
- **Tamper detection**: GCM auth tag failure → decryption raises; route returns 500 `VAULT_DECRYPT_FAILED` (never leaks which step failed).
- **No plaintext logs**: the word "plaintext" is not allowed in any log line. Value never hits stdout, stderr, or any log handler. Enforced by a simple grep test in CI.
- **Empty values prohibited**: zero-length plaintext rejected at the Pydantic layer.

## Env-var contract (hard, enforced)
Allowlist (everything else fails startup if it starts with `TENNETCTL_`):
```
TENNETCTL_VAULT_ROOT_KEY   TENNETCTL_MODULES   TENNETCTL_SINGLE_TENANT
TENNETCTL_APP_PORT         TENNETCTL_ALLOW_UNAUTHENTICATED_VAULT
```
Plus non-prefixed: `DATABASE_URL`, `LOG_LEVEL`.
Any `TENNETCTL_*` variable matching `SECRET|TOKEN|PASSWORD|PRIVATE_KEY|API_KEY` (case-insensitive) outside the allowlist → startup fails with a message pointing the user at the vault.

## Bootstrap-secret contract
On first boot after vault schema exists, the lifespan coroutine ensures the following keys exist; generates + inserts if missing:
- `auth.argon2.pepper` — 32 random bytes, base64-encoded
- `auth.session.signing_key_v1` — Ed25519 private key seed, base64-encoded
Both created with `created_by='sys'`, `description` explaining purpose, version=1. Idempotent — re-running against a populated vault is a no-op.

    **`crypto.py`** — pure functions, no globals:
    ```python
    from __future__ import annotations
    import base64
    import os
    from dataclasses import dataclass

    from cryptography.hazmat.primitives.ciphers.aead import AESGCM

    _KEY_BYTES = 32        # AES-256
    _NONCE_BYTES = 12      # GCM recommended

    @dataclass(frozen=True)
    class Envelope:
        ciphertext: bytes
        wrapped_dek: bytes   # wrap_nonce (12) || aesgcm(root).encrypt(wrap_nonce, dek)
        nonce: bytes         # data nonce (12)

    def load_root_key() -> bytes:
        b64 = os.environ.get("TENNETCTL_VAULT_ROOT_KEY")
        if not b64:
            raise RuntimeError(
                "TENNETCTL_VAULT_ROOT_KEY is required. "
                "Generate: python -c 'import os,base64; print(base64.b64encode(os.urandom(32)).decode())'"
            )
        try:
            raw = base64.b64decode(b64, validate=True)
        except Exception as e:  # noqa: BLE001
            raise RuntimeError("TENNETCTL_VAULT_ROOT_KEY is not valid base64") from e
        if len(raw) != _KEY_BYTES:
            raise RuntimeError(
                f"TENNETCTL_VAULT_ROOT_KEY must decode to {_KEY_BYTES} bytes; got {len(raw)}"
            )
        return raw

    def encrypt(plaintext: str, root_key: bytes) -> Envelope:
        if not plaintext:
            raise ValueError("vault rejects empty plaintext")
        dek = os.urandom(_KEY_BYTES)
        data_nonce = os.urandom(_NONCE_BYTES)
        ciphertext = AESGCM(dek).encrypt(data_nonce, plaintext.encode("utf-8"), None)
        wrap_nonce = os.urandom(_NONCE_BYTES)
        wrapped = AESGCM(root_key).encrypt(wrap_nonce, dek, None)
        return Envelope(ciphertext=ciphertext, wrapped_dek=wrap_nonce + wrapped, nonce=data_nonce)

    def decrypt(env: Envelope, root_key: bytes) -> str:
        wrap_nonce, wrapped = env.wrapped_dek[:_NONCE_BYTES], env.wrapped_dek[_NONCE_BYTES:]
        dek = AESGCM(root_key).decrypt(wrap_nonce, wrapped, None)
        plaintext = AESGCM(dek).decrypt(env.nonce, env.ciphertext, None)
        try:
            return plaintext.decode("utf-8")
        finally:
            # best-effort zeroing — Python bytes are immutable but we drop refs
            del plaintext, dek
    ```

    ```python
            self._ttl = ttl_seconds
            self._cache: dict[str, tuple[float, str, int]] = {}  # key → (fetched_at, value, version)

        async def get(self, key: str) -> str:
            now = time.monotonic()
            cached = self._cache.get(key)
            if cached and now - cached[0] < self._ttl:
                return cached[1]
            value, version = await self._fetch(key)
            self._cache[key] = (now, value, version)
            return value

        async def get_with_version(self, key: str) -> tuple[str, int]:
            value = await self.get(key)
            _, _, version = self._cache[key]
            return value, version

        def invalidate(self, key: str) -> None:
            self._cache.pop(key, None)

        def invalidate_all(self) -> None:
            self._cache.clear()

        async def _fetch(self, key: str) -> tuple[str, int]:
            row = None
            async with self._pool.acquire() as conn:
                row = await conn.fetchrow(
                    """SELECT ciphertext, wrapped_dek, nonce, version
                       FROM "02_vault"."10_fct_vault_entries"
                       WHERE key = $1 AND deleted_at IS NULL
                       ORDER BY version DESC LIMIT 1""",
                    key,
                )
            if row is None:
                raise VaultSecretNotFound(f"vault key not found: {key}")
            env = _crypto.Envelope(
                ciphertext=bytes(row["ciphertext"]),
                wrapped_dek=bytes(row["wrapped_dek"]),
                nonce=bytes(row["nonce"]),
            )
            return _crypto.decrypt(env, self._root_key), int(row["version"])
    ```

## Security notes

- **Root key lifecycle.** Root key is not rotatable in v0.2 (rotation = redeploy with new value + all secrets re-encrypted via a migration script). That's explicitly accepted in ADR-028 as the "basic but secure" baseline. v0.3 will add a proper two-key rotation ceremony.
- **No plaintext in logs / metrics / traces.** Grep test in Task 5 enforces this on every CI run. Service layer never logs the `value` argument. Audit metadata carries `key` name only.
- **Read audit vs node read.** HTTP `GET /v1/vault/{key}` audits every access (enterprise-grade visibility), but the in-process `vault.secrets.get` node does not — because that path runs inside every service call for OAuth secrets / Argon2 pepper / session key, and auditing each one would drown the log. The split is deliberate and documented in AC-5.
- **Tampering.** AES-GCM's authentication tag covers ciphertext + nonce + (optional) AAD. Any bit flip raises `InvalidTag`. Test 2 covers all three tamper surfaces.
- **Empty value defence.** Rejected at Pydantic + at crypto layer — double defence.
- **Key recycling.** In v0.2 a deleted key cannot be re-used. Prevents confusion where a rotated-then-deleted key silently reappears with a new meaning. v0.3 adds a "tombstoned" state with an admin-only forced-recycle.
- **No env leak.** The 5-item allowlist is strict; any mistake surfaces at startup, not at runtime.
