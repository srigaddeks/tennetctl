<objective>
## Goal
Ship PostHog-class analytics on top of the Audit Explorer: funnel analysis (multi-step event progression + conversion rates), cohort retention (weekly cohorts showing return rates), saved views (persist + reload filter presets), and CSV export. All backend + frontend + Robot E2E in one vertical slice.

## Purpose
The Audit Explorer (10-02) shows raw events. This plan adds the analytics layer: operators can trace how actors move through multi-step flows, see whether cohorts return, save their filter setups for reuse, and export event data. This is the "PostHog Funnels + Insights" equivalent.

## Output
- Migration 017: fct_audit_saved_views + dtl_audit_saved_view_details + v_audit_saved_views
- 5-file saved-views sub-feature (schemas/repo/service/routes/init)
- New endpoints on existing events routes: POST /v1/audit-events/funnel, GET /v1/audit-events/retention, ?format=csv on list
- Saved views CRUD: GET/POST/DELETE /v1/audit-saved-views
- 4 frontend additions: funnel-builder, retention-grid, saved-views-panel components + hooks
- Robot E2E covering funnel, saved views, CSV export
</objective>

<acceptance_criteria>

## AC-1: Funnel analysis endpoint
Given seeded events with actors progressing through steps A → B → C
When POST /v1/audit-events/funnel with {steps: ["A", "B", "C"]}
Then response has steps array: each step has event_key, users (distinct actors reaching step), conversion_pct (% from previous step)
And step[0].users = all actors who did A; step[1].users ≤ step[0].users (sequential progression)

## AC-2: Retention analysis endpoint
Given seeded events with actor_user_ids doing an anchor event and a return event across multiple weeks
When GET /v1/audit-events/retention?anchor=X&return_event=Y&bucket=week&periods=4
Then response has cohorts array: each cohort has cohort_period + cohort_size + retained array [int per period offset 0..N]
And cohort retention at period 0 = 100% (same-period as anchor)

      ```sql
      -- Uses date_trunc(bucket, created_at) to group cohorts
      -- Cohort = set of distinct actor_user_ids who did anchor in cohort_period
      -- For each cohort, count how many returned in period offset 0, 1, 2...N
      -- Use a CTE:
      WITH cohort_members AS (
        SELECT actor_user_id, date_trunc($bucket, created_at) AS cohort_period
        FROM "04_audit"."60_evt_audit"
        WHERE event_key = $anchor AND org_id = $org_id
        AND created_at >= NOW() - INTERVAL '1 period * periods'
      ),
      cohort_sizes AS (
        SELECT cohort_period, COUNT(DISTINCT actor_user_id) AS cohort_size
        FROM cohort_members GROUP BY cohort_period
      ),
      returns AS (
        SELECT c.actor_user_id, c.cohort_period,
          FLOOR(EXTRACT(EPOCH FROM (date_trunc($bucket, e.created_at) - c.cohort_period))
            / EXTRACT(EPOCH FROM INTERVAL '1 $bucket')) AS period_offset
        FROM cohort_members c
        JOIN "04_audit"."60_evt_audit" e ON e.actor_user_id = c.actor_user_id
        WHERE e.event_key = $return_event AND e.org_id = $org_id
        AND e.created_at >= c.cohort_period AND period_offset BETWEEN 0 AND $periods
      )
      ```
