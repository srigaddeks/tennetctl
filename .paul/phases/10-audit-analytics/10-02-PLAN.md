<objective>
## Goal
Ship the Audit Explorer — a PostHog-class event-analytics UI over the 10-01 backend. Filter bar (event-key glob, category, outcome, org, time range, metadata substring), cursor-paginated events table, detail drawer with pretty metadata + trace chip, stats mini-dashboard (by_event_key / by_outcome / by_category / time-series). Robot E2E + chrome-devtools live verification.

## Purpose
The audit event store is write-only and query-only without a UI. This plan delivers the "PostHog" equivalent — operators + developers can trace any action, filter by actor/org/event, and see aggregate activity without SQL. Full vertical per user directive: hooks + components + page + Robot E2E + chrome-devtools verify.

## Output
- 1 hooks file (useAuditEvents, useAuditEventDetail, useAuditEventStats, useAuditEventKeys)
- 4 components (filter-bar, events-table, event-detail-drawer, stats-panel)
- 1 page (/audit)
- Sidebar nav entry
- Robot E2E covering: navigate → filter by glob → cursor pagination → open detail → close → stats visible
- Chrome-devtools MCP walkthrough screenshot proving the page renders
</objective>
