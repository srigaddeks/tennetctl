<objective>
## Goal
OIDC / OAuth2 Authorization Code + PKCE SSO. Per-org IdP config (issuer, client_id, client_secret vault ref, scopes, claim mapping). JIT user provisioning with email_verified=true (IdP-attested).

## Purpose
OIDC is ubiquitous (Google Workspace, Microsoft Entra, Auth0, Keycloak). Enterprises that don't do SAML almost always do OIDC. Reusing `authlib` keeps this compact.
</objective>

## AC-2: Authorization Code + PKCE round-trip
```gherkin
Given OIDC config for Google Workspace on org acme
When user visits /v1/auth/oidc/acme/initiate
Then redirected to IdP authorize endpoint with PKCE code_challenge
When IdP redirects back to /v1/auth/oidc/acme/callback with authorization code
Then backend exchanges code + code_verifier for ID + access tokens
And validates ID token signature + iss + aud + exp
And extracts email/name/groups per claim mapping
And JIT creates/updates user + session
And audit iam.oidc.login_succeeded emitted
```

## AC-3: Failure modes
```gherkin
Given invalid state param OR tampered ID token
When callback fires
Then 403 with code=OIDC_VALIDATION_FAILED
And audit iam.oidc.login_failed emitted
```

## AC-4: Admin UI
```gherkin

Then admin sees per-org config form (issuer URL → auto-discover .well-known/openid-configuration, client_id, client_secret, scopes, claim mapping)
```
