<objective>
## Goal
SAML 2.0 SP-initiated SSO. Per-org IdP config (entity_id, SSO URL, x509 cert). python3-saml handles AuthnRequest + ACS Response validation. JIT user upsert (same email-match pattern as OIDC). Admin UI + pytest green.

## Purpose
Enterprises using ADFS, Okta, Azure AD SAML, or PingFederate require SAML. Completes SSO parity with Keycloak/Zitadel.
</objective>

```

```

## AC-3: SP metadata
```

```

## AC-4: SP-initiated flow
```
GET  /v1/auth/saml/{org_slug}/initiate → 302 to IdP SSO URL with signed AuthnRequest
POST /v1/auth/saml/{org_slug}/acs     → validate SAMLResponse, JIT upsert, session, 302 /
```

## AC-5: JIT user upsert
```
Email from NameID/attribute → match existing user across any account_type → create saml_sso user if new
```
