<objective>
## Goal
Dynamic groups whose membership is computed from rules over user attributes + email domains. Evaluator runs on user-create, attr-change, and email-verify. Orgs can also configure domain auto-join — email matching a domain triggers org membership + default group join.

## Purpose
Eliminates manual group assignment for "everyone in engineering" or "all @company.com users". Prerequisite for mapping SSO/SCIM groups cleanly.
</objective>

## AC-2: Rule evaluator
```gherkin
Given a group with rule kind=email_domain, rule_jsonb={value: "acme.com"}
When a user is created OR email_verified with email alice@acme.com
Then evaluator adds alice to the group via lnk_user_group
And emits iam.groups.auto_membership_added audit
When attribute-match rule {attr_def_id: X, op: "eq", value: "engineering"}
Then users with that attr auto-join
When rule is_active=false
Then no membership adjustment occurs
```

## AC-3: Domain auto-join
```gherkin
Given org O has dtl_org_email_domains {domain: "acme.com", auto_join: true} and default_group_id = G
When user alice@acme.com completes email verification
Then alice is added to lnk_user_org (org=O) if not already
And alice is added to lnk_user_group (group=G)
And audit iam.orgs.auto_joined emitted
```

## AC-4: Rule removal retracts membership
```gherkin
Given user is a rule-based member
When rule is revoked (is_active=false) or deleted
Then evaluator removes lnk_user_group entries that were rule-derived (marker column: source=rule vs source=manual)
```
