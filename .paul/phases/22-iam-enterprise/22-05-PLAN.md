<objective>
## Goal
Per-org CIDR allowlist. Middleware checks source IP against the session's org allowlist; empty = no restriction. Policy-gated enable/disable at org level.

## Purpose
Enterprises require "only corporate network can access this org". Protects against credential theft where attacker is off-network.
</objective>

## AC-2: Middleware enforces
```gherkin
Given session belongs to org O with non-empty allowlist [10.0.0.0/8, 192.168.1.0/24]
When a request arrives from source IP outside those ranges
Then middleware returns 403 with code=IP_NOT_ALLOWED
And audit iam.ip_allowlist.denied emitted with {org_id, source_ip, cidrs_checked}
When source IP is within any allowlisted CIDR
Then request proceeds
When allowlist is empty
Then middleware passes through (no restriction)
```

## AC-3: Admin CRUD
```gherkin
Given admin on /iam/security/ip-allowlist
Then admin sees CIDR list + Add/Remove actions
When Add submitted with invalid CIDR
Then UI shows validation error (bad CIDR, too broad e.g. 0.0.0.0/0 blocked with warning)
```

## AC-4: X-Forwarded-For handling
```gherkin
Given backend is behind a trusted reverse proxy
And policy `network.trusted_proxy_cidrs` is configured in vault
When a request arrives from a trusted proxy with X-Forwarded-For
Then middleware uses the left-most untrusted IP as the source
Else uses the direct socket IP
```
