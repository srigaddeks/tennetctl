<objective>
## Goal
Ship OIDC SSO as a full vertical: per-org provider config stored in DB (client secret in vault), OAuth2 authorization-code + PKCE flow, JIT user upsert on callback, admin UI to configure providers, and sign-in entry point.

## Purpose
OIDC is the most-used enterprise SSO method. Enables "Sign in with your company IdP" (Google Workspace, Okta, Azure AD, Auth0) without each org needing a separate integration. Keycloak/Zitadel/Clerk all lead with this.

## Output
- `20_oidc_sso` sub-feature: 5 Python files + migration + seed update
- Auth routes: GET /v1/auth/oidc/{org_slug}/initiate + GET /v1/auth/oidc/{org_slug}/callback
- Provider management routes: GET/POST/DELETE /v1/iam/oidc-providers
- Admin UI: /iam/security/sso — configure per-org OIDC provider
- Frontend: /auth/oidc/callback page + SSO entry on signin page
- pytest: 10+ tests covering CRUD, PKCE initiate, JIT upsert, error cases
</objective>

      enabled BOOLEAN NOT NULL DEFAULT true,
      created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
      updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
      deleted_at TIMESTAMP,
      UNIQUE (org_id, slug) WHERE deleted_at IS NULL
    View `v_oidc_providers`: SELECT p.*, o.slug AS org_slug
      FROM "03_iam"."30_fct_oidc_providers" p
      JOIN "03_iam"."10_fct_orgs" o ON o.id = p.org_id
      WHERE p.deleted_at IS NULL
    Include UP + DOWN.

    **Seed 02_dim_account_types.yaml**: add row id=5, code=oidc_sso, label="OIDC SSO".

    **schemas.py** (Pydantic v2):
    OidcProviderCreate: slug, issuer, client_id, client_secret_vault_key, scopes="openid email profile", claim_mapping={}
    OidcProviderRow: id, org_id, slug, issuer, client_id, client_secret_vault_key, scopes, claim_mapping, enabled, created_at

    **repository.py** (raw asyncpg, reads v_oidc_providers):
    - get_by_org(conn, org_id) -> list[dict]
    - get_by_org_slug(conn, org_slug, provider_slug) -> dict | None  (for auth routes)
    - create(conn, *, org_id, id, data) -> dict
    - soft_delete(conn, *, provider_id, org_id) -> None  (sets deleted_at + updated_at)

    **service.py** (pass conn to repo, pool only at route layer):
    - list_providers(pool, conn, ctx, org_id) -> list
    - create_provider(pool, conn, ctx, org_id, data) -> dict; emits audit iam.oidc.provider.created
    - delete_provider(pool, conn, ctx, provider_id, org_id); emits audit iam.oidc.provider.deleted
    - build_initiate_url(pool, conn, *, org_slug, provider_slug) -> str:
        Load provider via get_by_org_slug. Load client_secret via VaultClient (imported via importlib).
        PKCE: code_verifier = secrets.token_urlsafe(64); code_challenge = base64.urlsafe_b64encode(hashlib.sha256(verifier.encode()).digest()).rstrip(b'=').decode()
        State: HMAC-sign {org_slug, provider_slug, verifier, nonce=secrets.token_hex(16), exp=now+600} using vault key iam.oidc.state_secret (bootstrap default 32-byte random if missing).
        Use authlib.integrations.httpx_client.AsyncOAuth2Client to build authorization_url with code_challenge_method=S256.
        Return redirect URL string.
    - handle_callback(pool, conn, ctx, *, org_slug, code, state) -> (user_dict, token):
        Decode + validate HMAC state (check exp, org_slug match).
        Load provider, exchange code for tokens with code_verifier from state.
        Verify ID token: check iss matches provider.issuer, aud matches client_id, exp not passed.
        Extract email via claim_mapping["email"]. Extract display_name via claim_mapping.get("name").
        JIT upsert: SELECT from v_users WHERE email=$1 AND deleted_at IS NULL.
          If found: use existing user_id (any account_type).
          If not found: INSERT into fct_users with account_type_id=5 (oidc_sso), uuid7 id.
        Mint session via existing sessions repo (import via importlib).
        Emit audit iam.oidc.sso.signin with metadata={user_id, org_slug, provider_slug, is_new_user}.
        Return (user_row, session_token).

    **routes.py** (provider management, require session):
    GET  /v1/iam/oidc-providers -> list for current org
    POST /v1/iam/oidc-providers -> create
    DELETE /v1/iam/oidc-providers/{provider_id} -> soft delete

    **Add to backend/02_features/03_iam/routes.py** (auth routes, no session required):
