<objective>
## Goal
Extend `tennetctl-py` with 4 new capability modules: `client.flags`, `client.iam`, `client.audit`, `client.notify`. Layer on the Transport + error hierarchy from 28-01 — no new plumbing.

## Purpose
After this plan ships, the Python SDK covers the core business-workflow surface of the platform. Admin UIs, backend services, and external callers can all use one client for the things they reach for 80% of the time.

## Output
- 4 new modules importable via `client.flags/iam/audit/notify`
- `flags.evaluate` with 60s SWR cache keyed on `(flag_key, entity_hash)`
- `iam` read-only helpers: `users`, `orgs`, `workspaces`, `roles`
- `audit` query helpers (events list/get/stats/tail — emission is node-only on backend so no SDK emit method)
- `notify.send` transactional wrapper with optional `Idempotency-Key` header
- ≥80% coverage on each new module
</objective>
