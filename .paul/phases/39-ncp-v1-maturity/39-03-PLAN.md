<objective>
## Goal

Ship a working v1→v2 node version bump on `iam.orgs.get` so the protocol's "register two keys + deprecated_at + replaced_by" pattern from §13 is exercised end-to-end. Document the migration pattern in a new ADR so future version bumps follow the same recipe.

## Purpose

NCP v1 §13 promises parallel v1/v2 coexistence via paired keys, but no node has actually been through the pattern. Until someone does it, the "escape hatch" is hypothetical — every claim that v1 can evolve without a hard cutover is unverified.

Pick `iam.orgs.get` (control, simple, low-risk) and add `iam.orgs.get_v2` that returns `{org, workspace_count}` — a breaking Output-schema change (new required field = breaking per §13). v1 stays live + marked deprecated. No callers break.

## Output

- New node `iam.orgs.get_v2` with Output `{org: dict | None, workspace_count: int}`
- v1 `iam.orgs.get` marked `deprecated_at` + `replaced_by: iam.orgs.get_v2` in manifest
- ADR-032 documents the v1→v2 recipe (copy-and-adapt for future bumps)
- NCP v1 §13 gets a "See ADR-032 for the first real migration" footnote + small changelog entry
</objective>

## AC-2: Migration pattern documented

```gherkin
Given a new ADR at 03_docs/00_main/08_decisions/032_v1_to_v2_versioning_pattern.md
When a contributor needs to bump a node version
Then the ADR walks through the recipe using iam.orgs.get → _v2 as the worked example:
  1. Copy the node class to <name>_v2.py with the new Input/Output shape + version=2 in the class
  2. Register the new key in the manifest alongside the old
  3. Add deprecated_at + replaced_by on the v1 entry
  4. (Later) Callers migrate one at a time
  5. (Future phase) Remove v1 after `deprecated_at + N days` elapses, emit tombstoned_at
And §13 of the NCP spec links to ADR-032 as the canonical example
```

      - **Decision:** Paired-key pattern with `_v<N>` suffix. v1 stays live + marked `deprecated_at + replaced_by`. Callers opt in to v2 by calling the new key. Eventual cleanup (tombstone of v1) is a separate later step after `deprecated_at + N days`.
